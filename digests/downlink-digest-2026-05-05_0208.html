Hackers have been actively exploiting a critical unauthenticated remote code execution vulnerability, CVE-2026-22679, in the Weaver E-cology office automation platform since mid-March. This enterprise software, which is primarily utilized by Chinese organizations for workflows, document management, and HR processes, is affected in all 10.0 builds released before March 12. This architectural flaw enables attackers to pass crafted values that execute as system commands on the server. Threat intelligence researchers at Vega analyzed the malicious activity and reported that the exploitation campaign lasted approximately a week. The attacks commenced five days after the software vendor published a security update and two weeks before the vulnerability was disclosed to the public.

Vega detailed several distinct phases of the attack, beginning with researchers checking for RCE capabilities by triggering ping commands from the Java process to a Goby-linked callback. Following this initial probe, the threat actors attempted multiple PowerShell-based payload downloads, but all were successfully blocked by endpoint defenses. The attackers then tried to deploy a target-aware MSI installer named fanwei0324.msi, but the execution failed and no further activity was observed.

After these unsuccessful attempts, the threat actors reverted to the RCE endpoint, utilizing obfuscated and fileless PowerShell to repeatedly fetch remote scripts. Throughout all phases, the actors executed standard reconnaissance commands, including whoami, ipconfig, and tasklist. Despite having successful remote command execution capabilities, the attackers never established a persistent session on the targeted systems.

Vega explained that every observed malicious process was parented by java.exe, which is the Tomcat-bundled Java Virtual Machine used by Weaver, and operated without preceding authentication. Vulnerability Overview

Hackers have been exploiting a critical unauthenticated remote code execution flaw, identified as CVE-2026-22679, in the Weaver E-cology office automation platform since mid-March. The software serves as an enterprise collaboration tool for workflows, document management, HR, and internal business processes, with primary usage concentrated among Chinese organizations. The vulnerability impacts all Weaver E-cology 10.0 builds released prior to March 12.

The root cause is an exposed debug API endpoint that improperly allows user-supplied parameters to reach backend Remote Procedure Call functionality without authentication or input validation. This design flaw effectively turns the endpoint into a remote command execution interface, allowing attackers to run system commands directly on the server.
