Malware Categories and Case Studies

The analysis categorizes the malicious extensions into six distinct operational types based on their primary objectives. Each category represents a unique attack vector that bypasses traditional network security controls. The Chrome MCP Server extension disguises itself as an AI browser automation tool. It establishes a persistent WebSocket connection to a remote command and control server. Once connected, it accepts over 30 remote commands, including arbitrary JavaScript execution and HTTPS traffic interception via the Chrome Debugger Protocol.

Supersonic AI markets itself as an email assistant but functions as a data exfiltration tool. It uses content scripts to collect comprehensive email data and sends it to an external server in plaintext. This approach captures all account emails and exposes sensitive information like one-time passwords. Reverse Recruiting targets AI API keys and user profile information. When users provide OpenAI, Gemini, or Claude API keys, the extension forwards them to a backend server in custom HTTP headers. It also syncs personal data, including names, emails, and salary expectations, to a remote endpoint.

Chat AI for Chrome modifies default search engine settings to route queries through attacker-controlled domains. It maintains persistent cross-device tracking by storing unique identifiers across cookies, local storage, and Chrome sync storage. The extension automatically recreates deleted tracking cookies to ensure continuous monitoring. Certain extensions impersonate popular graphics brands to exploit user trust. They utilize the onInstalled listener to force a thank-you page immediately after installation. This page redirects traffic to third-party domains to drive unwanted browser installations.

A Chinese-language translation extension requests permissions far exceeding its stated functionality. It registers listeners for every completed HTTP request and dynamically fetches proxy auto-configuration scripts. This allows operators to selectively route user traffic through arbitrary proxy servers without submitting extension updates.

Technical Techniques and Capabilities

Browser extensions operate within the browser’s trusted process with user-granted permissions. They are given privileged access to web content and network requests. The research documents several recurring techniques used by these malicious extensions:

  • WebSocket-based C2 Channels: Extensions maintain persistent bidirectional communication for command dispatch. These connections automatically reconnect on network interruptions and persist across browser restarts without requiring process injection.
  • Browser API Hooking: Malicious scripts intercept JavaScript API calls before network transmission. They replace native functions like window.fetch or XMLHttpRequest to operate in a JavaScript context before data is encrypted.
  • DOM-based Exfiltration: Extensions extract page content through passive observation rather than network interception. They read rendered page content directly, generating no network requests for data collection and bypassing network-level security controls entirely.
  • Dynamic Proxy Configuration: Extensions download and apply remote proxy configuration scripts. This enables selective traffic routing per domain or URL pattern without requiring OS-level permissions or store approval.
  • Cross-Storage Persistence: Malware stores identifiers across multiple browser APIs with automated recreation upon deletion. They monitor storage-change events and sync identifiers across devices signed into the same account.
  • Misuse of One-Time Events: Extensions install payloads via the chrome.runtime.onInstalled listener. This allows code execution once during installation or update, firing before the user interacts with the extension.

AI-Accelerated Campaigns

The article documents an increasing trend of threat actors leveraging large language models to accelerate malware production. One example is the 10xprofit affiliate hijacking campaign, which runs six extensions that silently inject affiliate tags into popular online retailers. All six extensions bear AI-generated code fingerprints, including formulaic comments, identical structures, and template-based scaffolding. This demonstrates how generative AI lowers the barrier to entry for deploying multiple malicious extensions targeting different platforms.

Mitigation and Palo Alto Networks Protection

Defending against these threats requires treating the browser as a primary enterprise attack surface. Palo Alto Networks recommends that organizations and users source extensions only from trusted providers. Users must scrutinize requested permissions and adhere to the principle of least privilege. Granting broad access to browser data can authorize the interception of sensitive credentials and proprietary session information. Palo Alto Networks customers are protected through Advanced URL Filtering, Advanced DNS Security, Prisma Browser, Prisma AIRS, and Advanced WildFire. These products identify malicious domains, monitor extension behavior in real time, and enforce secure usage policies across AI applications.
