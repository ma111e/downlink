Incident Overview

A malicious Hugging Face repository impersonating OpenAI’s Privacy Filter project distributed an information-stealing malware payload to Windows users. The repository briefly reached the number one position on the platform’s trending list and accumulated 244,000 downloads before being removed following user reports. Hugging Face serves as a platform for developers and researchers to share AI models, datasets, and machine learning tools. The malicious repository exploited this ecosystem by hosting a pre-trained AI system that combined weight files, configuration data, and malicious code.

Discovery and Impersonation Tactics

Researchers at HiddenLayer discovered the campaign on May 7 after identifying a repository named Open-OSS/privacy-filter. The threat actors typosquatted OpenAI’s legitimate Privacy Filter release by copying its model card nearly verbatim. The repository shipped a loader.py file that was designed to fetch and execute infostealer malware on Windows machines. This Python script included fake AI-related code to appear harmless during initial inspection.

Malware Execution Chain

The loader.py script operated by disabling SSL verification and decoding a base64-encoded URL pointing to an external resource. It then fetched and executed a JSON payload containing a PowerShell command. This command executed in an invisible window to download a batch file named start.bat. The batch file performed privilege escalation, downloaded the final payload known as sefirah, added it to Microsoft Defender’s exclusions, and executed it. The final payload is a Rust-based infostealer designed to harvest sensitive data from the compromised system.

Targeted Data and Exfiltration

The infostealer targeted the following sensitive information:

  • Browser data from Chromium- and Gecko-based browsers, including cookies, saved passwords, encryption keys, browsing data, and session tokens
  • Discord tokens, local databases, and master keys
  • Cryptocurrency wallets and wallet browser extensions
  • SSH, FTP, and VPN credentials and configuration files, including FileZilla
  • Sensitive local files and wallet seeds or keys
  • System information and multi-monitor screenshots

All stolen data was compressed and exfiltrated to a command-and-control server located at recargapopular[.]com. The malware also incorporated extensive anti-analysis features to evade detection. These features included checks for virtual machines, sandboxes, debuggers, and analysis tools.

Campaign Metrics and Related Infrastructure

The exact number of victims remains unclear due to suspected artificial inflation of engagement metrics. Researchers noted that the vast majority of the 667 accounts that liked the repository appear to be auto-generated. The 244,000 download count was also likely inflated through similar automated means. By examining these metrics, researchers uncovered other repositories that utilized the same malicious loader infrastructure. HiddenLayer also identified technical overlaps with an npm typosquatting campaign that distributed the WinOS 4.0 implant.

Remediation Recommendations

Users who downloaded files from the malicious repository are advised to take immediate action. They should reimage their machines to ensure complete removal of the threat. All stored credentials must be rotated to prevent unauthorized access. Cryptocurrency wallets and seed phrases should be replaced to protect digital assets. Finally, browser sessions and tokens must be invalidated to block persistent access. Threat actors have previously abused Hugging Face to host malicious models despite the platform’s existing security measures.

  • The malware bypasses standard security controls by disabling SSL verification and executing a PowerShell command in an invisible window to download and run the final payload.
  • The threat actor reused infrastructure and tactics across campaigns, as the same loader infrastructure was found in other repositories and overlapped with an npm typosquatting campaign distributing the WinOS 4.0 implant.
  • Platform popularity metrics were weaponized, with the malicious repository artificially inflating its 244,000 downloads and 667 likes through auto-generated accounts to gain traction on Hugging Face's trending list.
  • The infostealer's Rust-based architecture and extensive anti-analysis checks for VMs, sandboxes, and debuggers indicate a deliberate focus on evading automated security research environments.

cPanel and WHM have issued urgent security patches to resolve three newly disclosed vulnerabilities that pose significant risks to server environments. The first vulnerability, identified as CVE-2026-29201, has a CVSS score of 4.3 and results from inadequate input validation within the feature::LOADFEATUREFILE adminbin call.

This flaw could allow an attacker to perform arbitrary file reads. The second vulnerability, CVE-2026-29202, carries a high CVSS score of 8.8 and stems from insufficient validation of the plugin parameter in the create_user API call.

This issue enables arbitrary Perl code execution under the context of an already authenticated system user. The third vulnerability, CVE-2026-29203, also holds a CVSS score of 8.8 and involves unsafe symlink handling.

This defect permits users to alter access permissions on arbitrary files via chmod, potentially leading to denial-of-service attacks or privilege escalation. To address these issues, cPanel has released updated versions across multiple release branches.

Administrators running cPanel and WHM must update to 11.136.0.9 or higher, 11.134.0.25 or higher, 11.132.0.31 or higher, 11.130.0.22 or higher, 11.126.0.58 or higher, 11.124.0.37 or higher, 11.118.0.66 or higher, 11.110.0.116 or higher, 11.110.0.117 or higher, 11.102.0.41 or higher, 11.94.0.30 or higher, or 11.86.0.43 or higher. Users operating WP Squared must update to version 11.136.1.10 or higher.

Additionally, version 110.0.114 has been deployed as a direct update for customers still utilizing CentOS 6 or CloudLinux 6. Although there is no current evidence of these vulnerabilities being actively exploited, the announcement arrives just days after CVE-2026-41940 was weaponized as a zero-day.

That previous flaw was used to distribute Mirai botnet variants and a ransomware strain known as Sorry. Security teams are strongly advised to apply the latest updates immediately to ensure optimal protection.

Vulnerability Overview

cPanel and WHM have released critical security updates to address three distinct vulnerabilities that could enable privilege escalation, arbitrary code execution, and denial-of-service attacks. Each vulnerability has been assigned a unique CVE identifier and CVSS score:

  • CVE-2026-29201 (CVSS 4.3): This flaw involves insufficient input validation of the feature file name within the “feature::LOADFEATUREFILE” adminbin call. Exploitation could result in an arbitrary file read.
  • CVE-2026-29202 (CVSS 8.8): This vulnerability stems from insufficient input validation of the “plugin” parameter in the “create_user API” call. It allows for arbitrary Perl code execution on behalf of an already authenticated account’s system user.
  • CVE-2026-29203 (CVSS 8.8): This issue involves unsafe symlink handling that permits a user to modify access permissions of an arbitrary file using chmod. This could lead to denial-of-service or possible privilege escalation.

EXEC Executive Overview

This digest covers two distinct but thematically linked security developments reported during the May 8–9, 2026 coverage window. The first centers on a platform abuse campaign targeting the Hugging Face ecosystem, while the second addresses critical vulnerability disclosures in the cPanel and WHM administrative suite. Both incidents highlight how threat actors continue to leverage widely trusted software repositories and hosting control panels to facilitate malware distribution and privileged access.

In the Hugging Face incident, adversaries published a malicious repository impersonating OpenAI’s Privacy Filter project. Prior to platform removal, the repository distributed a Rust-based infostealer targeting Windows workstations. The malware’s loader script executed PowerShell commands to download and run a secondary payload designed to extract browser credentials, Discord authentication tokens, cryptocurrency wallet seed phrases, and system-level credentials. Exfiltrated data is compressed and transmitted to an external command-and-control infrastructure. The infostealer incorporates anti-analysis techniques to hinder static and dynamic reverse engineering. Intelligence indicates that the repository’s high download metrics and community engagement were artificially inflated using auto-generated accounts, a tactic consistent with known platform manipulation strategies aimed at bypassing trust-based discovery mechanisms.

Separately, cPanel and WHM released updates addressing three newly disclosed vulnerabilities: CVE-2026-29201, CVE-2026-29202, and CVE-2026-29203. These flaws stem from insufficient input validation and unsafe symbolic link handling within the control panel software. Successful exploitation could result in arbitrary file read access, arbitrary Perl code execution, and privilege escalation through symlink manipulation. The updates also address denial-of-service conditions triggered by malformed inputs. Patched releases are designated as cPanel and WHM version 11.136.0.9 or higher, WP Squared version 11.136.1.10 or higher, and version 110.0.114 for legacy CentOS 6 and CloudLinux 6 environments. While no active exploitation of these specific CVEs has been confirmed, the disclosures follow a broader trend of cPanel vulnerabilities being weaponized to deliver botnet variants and ransomware payloads in previous threat cycles.

Together, these reports underscore a persistent operational pattern: adversaries prioritize platforms and administrative interfaces with broad reach and high user trust. Whether through fabricated repository listings or unpatched host control panel flaws, the underlying mechanisms enable credential harvesting, privilege escalation, and persistent access. The technical details across both incidents reflect a continued emphasis on evasion techniques, supply-chain-adjacent distribution vectors, and rapid exploitation of widely deployed software ecosystems.
