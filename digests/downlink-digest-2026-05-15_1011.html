    Vulnerability Overview

    Microsoft has disclosed CVE-2026-42897, a critical security defect impacting on-premises versions of Exchange Server.

    The vulnerability carries a CVSS score of 8.1 and is officially classified as a spoofing bug rooted in a cross-site scripting flaw.

    An anonymous researcher discovered and reported the issue to Microsoft.

    The vendor describes the defect as an improper neutralization of input during web page generation that allows unauthorized attackers to perform spoofing over a network.

    Exploitation Mechanics

    Microsoft has tagged the vulnerability with an “Exploitation Detected” assessment, confirming active real-world abuse.

    The attack vector involves an attacker sending a crafted email to a target user.

    When the recipient opens the message in Outlook Web Access, arbitrary JavaScript code can be executed within the web browser context.

    This execution is not automatic and requires the victim to interact with the email under specific, additional conditions outlined by the vendor.

    Affected Systems and Scope

    The vulnerability impacts the following on-premises Exchange Server versions at any update level:

    • Exchange Server 2016
    • Exchange Server 2019
    • Exchange Server Subscription Edition

    Microsoft explicitly states that Exchange Online is not affected by this vulnerability.

    The scope of the active exploitation remains unclear, as the company has not disclosed the identity of the threat actor, the specific exploitation techniques used, or the scale of the campaign.

    It is also unknown whether any specific targets have been successfully compromised.

    Mitigation Strategies

    Microsoft is providing immediate protection through the Exchange Emergency Mitigation Service.

    This service is enabled by default and automatically applies a URL rewrite configuration to block the vulnerability.

    Administrators are advised to ensure the Windows service is running if it is not already active.

    For environments with air-gap restrictions that prevent service deployment, Microsoft recommends using the Exchange on-premises Mitigation Tool.

    The tool can be downloaded from a Microsoft-provided URL and executed via an elevated Exchange Management Shell.

    Single servers can be patched using a direct command, while all internal servers can be remediated simultaneously through a pipeline script.

    Known Issues and Recommendations

    Microsoft has documented a cosmetic bug in the mitigation tool where the description field incorrectly displays “Mitigation invalid for this exchange version.”

    The vendor clarifies that this message is purely cosmetic and does not indicate a failure, provided the status field shows “Applied.”

    The Exchange Team is actively investigating how to correct this display error.

    In the absence of detailed threat intelligence or a permanent patch, Microsoft strongly recommends that all administrators immediately apply the available mitigations to secure their on-premises Exchange environments.
