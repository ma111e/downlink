In plain words

An attacker took control of the simonecorsi/mawesome GitHub project on June 24, 2026, changed its code, and moved several saved version labels so they pointed to the attacker’s code. Anyone whose GitHub automated work used those labels after that time may have run the attacker’s code without realizing it. This matters because a widely used shared project can become a way to reach many other people’s software-building processes at once.

Summary
The simonecorsi/mawesome GitHub Action was compromised after an attacker force-pushed malicious code and repointed version tags, causing workflows using those tags to run attacker-controlled code. Users should treat affected runs after June 24, 2026 as potentially compromised.
  • The compromise affected users through version tags, showing that workflows referencing those tags could run changed code even if the workflow file itself was not modified.
  • Because several version tags were repointed to the malicious commit, the impact was not limited to a single tagged release of simonecorsi/mawesome.
  • The article links this incident to a very similar GitHub Actions compromise involving codfish/semantic-release-action earlier the same day, indicating more than one comparable event was reported on June 24, 2026.

Google releases new privacy controls for activity history, personalization
bleepingcomputer.com · 24 Jun 2026 23:46 UTC openai-codex / gpt-5.5
TAGSgooglegoogle-accountgoogle-flightsgoogle-hotelsgoogle-lensgoogle-mapsgoogle-newsgoogle-playgoogle-searchgoogle-shoppinggoogle-translatesearch-liveweb-and-app-activity

In plain words

Google is adding new account settings that separately control whether it saves your activity in Search services and Google Play, and whether it uses that information to personalize what you see. People with Google accounts are affected, especially if they already have Web & App Activity turned on, because the new Search history and media-saving option may be turned on after the change. This matters because saved information can include searches, Maps activity, images, files, audio, and video from tools like Lens or voice interactions, and Google says it may use that data to improve its services, including AI models.

Summary
Google is rolling out separate controls for Search and Google Play history and personalization, giving users more granular privacy settings. If Web & App Activity is enabled, the new Search Services History and its Save Media option may be turned on automatically, so users should review the settings.

Google is rolling out new privacy controls for Search services and Google Play that separate saved history from personalized recommendations. The company said Search services include Search, Maps, Shopping, Hotels, Flights, Translate, and News. Users will see the change in their Google Account over the next few days.

The new Search Services History setting will control whether activity from Search services is saved to a user’s Google Account, while Personalized Recommendations will control whether Google uses saved data to tailor the experience. If Web & App Activity is currently turned on, Google says Search Services History will be turned on after the transition, and the Save Media subsetting will also be turned on. Google says changes to Web & App Activity will be separate from the new Search services history and personalization settings.

Saved media can include images, files, audio, and video from interactions with Search services, such as visual searches with Google Lens or audio from voice-based interactions. Google says saved media may be used to develop and improve Google services and technologies, including AI models and safety measures. Users can turn off Save Media at any time, delete individual pieces of media from history, and adjust or delete activity settings for Search services and Google Play.

  • Users who previously enabled Web & App Activity should specifically review the new settings because Google says the Save Media option will be turned on automatically after the transition.
  • The new separation means users can choose to keep Search services history for convenience while disabling Personalized Recommendations so that saved data is not used to tailor what they see.
  • Google says prior auto-delete choices from Web & App Activity will carry over to Search Services History and Play History, so users do not have to recreate those retention limits unless they want to change them.
  • The media-saving change is especially important for people using Lens or voice-based Search features because Google says saved media can include images, files, audio, and video from those interactions.
Key Points
  • 01Google is rolling out new privacy controls for Search services and Google Play that separate saved history from personalized recommendations.
  • 02Search services covered by the new settings include Search, Maps, Shopping, Hotels, Flights, Translate, and News.
  • 03If Web & App Activity is currently turned on, Google says Search Services History and the Save Media subsetting will also be turned on after the transition.
  • 04Saved media can include images, files, audio, and video from interactions with Search services, and Google says it can be used to develop and improve services and technologies, including AI models and safety measures.
  • 05Google says users can turn off the Save Media option, delete individual pieces of media from their history, and adjust or delete their activity settings at any time.
