Overview

Mitiga Labs published research on risks in AI agent instruction files and released Skillgate, a free scanner for detecting risky or malicious patterns in AI skills and configurations. The research covered a six-month investigation into files that AI agents and coding assistants read and trust, including Cursor rules, Anthropic Skills, Claude Hooks, AGENTS.md and CLAUDE.md files, MCP server configs, package.json droppers, and hidden bytecode artifacts.

The article’s central claim is that instruction files can function like malware because the agent follows them automatically, often with near-zero validation. Mitiga Labs says it found prompt-exfiltration tradecraft in the wild, attacker-controlled ANTHROPIC_BASE_URL overrides routing Claude traffic through man-in-the-middle proxies, permission-bypass defaults, and more than 1,230 hardcoded API keys and JWT tokens.

Instruction File Surface

Mitiga Labs mapped several categories of AI-agent instruction and configuration files that it says matter to modern coding assistants and agents:

  • Skills, such as MD files that provide task-specific context and instructions.
  • Claude Hooks, which are JSON-defined bash commands fired on Claude Code triggers.
  • Agent brain files such as AGENTS.md and CLAUDE.md.
  • MCP server configs, including settings.json or mcp.json blocks.
  • Agentic rules and tool configs such as .cursorrules, aider YAML, opencode, Continue, and equivalents.
  • Supply-chain droppers such as package.json files placed in an agent’s working directory.
  • Poisoned bytecode such as .pyc files hidden inside agent-tool directories.

The article states that the list of relevant instruction-file types continues to grow.

Research Objective and Attack Chain

Mitiga Labs says its earlier research showed that AI agents follow these files with near-zero validation and full trust. The article states that a legitimate-looking GitHub repository with instructions already staged is the easiest delivery vector demonstrated in its recent work.

The strongest chain described in the article combines several steps:

  • An npm install occurs.
  • An approved project path is injected.
  • The developer launches the AI inside that project.
  • A Claude Hook fires a dropper.
  • A Skill is invoked on a heartbeat cadence.
  • Each Skill can chain to the next to pull a fresh dropper or execute the operator’s objective.

Mitiga Labs presents this as a path from npm install to persistent AI agent compromise.

Skillforge and Skillgate

The article describes two paired tools developed during the research.

Skillforge is an internal Mitiga Labs web application that forges malicious skills. It is built from public research, Mitiga’s red-team work, and adversarial behavior mapped on MITRE ATT&CK and adjacent frameworks.

Skillgate is the community-facing scanner released by Mitiga Labs. It detects malicious or risky techniques inside skills and other instruction files. Its rule set is fed by internal research, Skillforge-generated samples, public research, and incidents observed in the wild.

Mitiga Labs says the two systems operate as a closed learning loop in which adversarial skills are generated, scanned, and used to improve Skillgate’s detection rules. The article says this cycle has run on a near-daily basis for four to five months.

Skillgate Capabilities

According to the article, Skillgate currently provides simultaneous scanning across skills, hooks, agent brain files, MCP configs, agentic rules, supply-chain droppers, and bytecode artifacts. It has scanned more than 50,000 files across more than 7,000 distinct public repositories.

The scanned corpus came from GitHub, arXiv-referenced samples, and Skillforge-generated adversarial inputs. Skillgate contains more than 80 detection rules organized into technique families. It also includes the Gator Agent, an LLM-based reviewer that runs a second pass on flagged artifacts and was iterated against thousands of real and synthetic samples.

Detection Technique Families

The article lists several technique families covered by Skillgate:

  • Direct execution and configuration abuse, including shell commands in hooks, dangerous settings toggles, permission-escalation patterns, and supply-chain droppers.
  • Prompt manipulation and instruction subversion, including prompt injection, jailbreaks, trigger phrases, instruction overrides, prompt leakage, and prompt exfiltration.
  • Tool and MCP server poisoning, targeting the agent’s tool layer.
  • Supply-chain and distribution attacks, including poisoned bytecode, external-binary indirection, npm post-install droppers, and configurations that add untrusted sources.
  • Obfuscation and evasion, including Unicode tricks, homoglyph substitutions, encoding chains, and steganography.
  • Credential exposure and data exfiltration, including hardcoded provider keys, secret leakage through logging or telemetry-shaped instructions, file-system exfiltration patterns, and instructions to send data somewhere it should not go.

Mitiga Labs says each rule is documented in the platform and backed by at least one public CVE, named research disclosure, Skillforge-generated synthetic sample, or artifact captured in the wild.

In-the-Wild Findings

Prompt Exfiltration

Mitiga Labs identifies prompt exfiltration as one of the most consequential classes it tracks. The article describes it as a malicious instruction file telling the agent to capture the developer’s prompts and quietly send them to an operator-controlled endpoint.

Mitiga Labs says it caught a live instance of this technique in the wild, but it is holding the full breakdown for a dedicated report. The article states that Skillgate already detects the technique class.

Supply-Chain Trust Through External URLs

The article shows Skillgate detecting a file that grants an agent permission to use cURL against an external address. Mitiga Labs says a dependency may be benign at the time of review, but a configuration that allows the agent to fetch and execute content from a URL can become an arbitrary code-execution primitive if that URL is compromised or controlled by an untrusted operator.

MCP Proxying via Hardcoded IP Addresses

Mitiga Labs shows Skillgate flagging an MCP server endpoint set to a hardcoded IP address. The article presents this as a sign of possible man-in-the-middle proxying.

YOLO Mode and ANTHROPIC_BASE_URL Overrides

The article highlights a configuration pairing YOLO mode with an ANTHROPIC_BASE_URL override. Mitiga Labs says the issue is not Anthropic’s own endpoint, but rather that the override can redirect all Claude API traffic from the developer’s machine through a third-party host.

The article states that such a third-party host would be positioned to log, replay, or modify every conversation. It also shows Skillgate detections for change modification being permitted, PowerShell execution, and policy bypass in a SKILL.md file.

Affected Roles and Environments

Mitiga Labs says AI coding assistants have made many roles into part-time developers, increasing the threat surface beyond traditional software engineering teams. The article says manually verified in-the-wild findings came from roles including:

  • DevOps engineers at security companies.
  • Researchers at data-platform companies.
  • AI engineers at web3 startups.
  • Full-stack developers at fintech companies.
  • Founders and CEOs of marketing platforms.
  • PhD students in data-related programs.
  • Project and product managers.

The article divides verified findings into two broad patterns: AI configuring infrastructure and AI configuring environment.

Infrastructure and Environment Configuration Risks

For infrastructure configuration, Mitiga Labs shows Skillgate flagging a service bound to 0.0.0.0 alongside hardcoded Postgres credentials. The article states that pairing these creates a compromise primitive consisting of a publicly bound service plus a credential pair, gated only by an inbound firewall rule.

For environment configuration, the article shows Skillgate flagging hardcoded provider API keys exposed in AI-agent configuration files. Provider categories mentioned include Anthropic Claude, OpenAI, Google Gemini, xAI Grok, Chinese-vendor models including MiniMax and DeepSeek, Context7, Databricks, Supabase, Vercel, YouTube, Langsmith, and Google Cloud Storage signed URLs and bucket API keys.

Mitiga Labs notes that flipping a repository to private after exposure is not enough because public mirrors and code-search indexes may already have cached the commits and embedded tokens. The article recommends rotating every applicable key, auditing for unauthorized use, and then addressing repository visibility.

Mitiga Labs states that it did not access, use, or attempt to authenticate with any discovered credential and treated all keys as in scope only for responsible disclosure.

Impact and Blast Radius

The article says Mitiga Labs did not weaponize, propagate, or test any discovered tradecraft. Based on what it observed, it ranks realistic impact as follows:

  • Supply-chain compromise, described as the highest-value outcome for an adversary because a poisoned instruction file inside a popular agent template can reach every developer who clones it.
  • Unauthorized access through MITM proxying, based on observed base-URL overrides and IP-pinned MCP endpoints.
  • Stolen credentials, with potential direct financial impact through provider wallet abuse and direct data impact through ingested data or tenant access.

Mitiga Labs characterizes these as early outcomes in a broader emerging era.

Methodology and Limits

The article states that the corpus included more than 50,000 instruction files across more than 7,000 distinct public repositories scanned from April to June 2026. Sources included public GitHub repositories containing recognized instruction-file patterns, arXiv-referenced skill samples, and Skillforge-generated adversarial inputs.

Mitiga Labs says every in-the-wild finding cited in the article was manually triaged by the Gator Agent and a human reviewer. It also states that raw scanner output is not published as a finding.

The article notes that Skillgate flags configurations that are risky in context, and some YOLO-mode or cURL-allowlist findings may be intentional choices in trusted environments. It advises readers to treat each flag as requiring review rather than as proof of malware.

Mitiga Labs says it did not measure actual exploitation, did not honeypot, contact, or test operator infrastructure, and went no further than identifying repository owners for disclosure.

Recommended Workflow

Mitiga Labs recommends that teams using AI-agent instruction files take several steps:

  • Run Skillgate against instruction-file paths added to source control, including .cursorrules, .claude/, CLAUDE.md, AGENTS.md, .continue/, mcp.json, and post-install scripts in the agent’s working directory.
  • Block merges if Skillgate raises findings in the Direct Execution, Tool and MCP Poisoning, or Credential Exposure families until a human reviewer signs off.
  • Re-scan on every model or agent upgrade because new runtime permissions may ship quickly.
  • For external repositories, run Skillgate before cloning or opening the files in an editor.

The article states that Skillgate is free at skillgate.mitiga.ai and that public scans can be browsed anonymously, while submitting a skill or repository for scanning requires a free account.

  • AI agent compromise can occur without traditional malware because a malicious instruction file can make the trusted agent itself capture prompts, read files, or send data externally.
  • A base-URL override such as ANTHROPIC_BASE_URL can turn a normal Claude configuration into a man-in-the-middle risk by routing all Claude API traffic through a third-party host.
  • Repository privacy changes are insufficient after secrets are exposed because public mirrors and code-search indexes may already have cached the commits and embedded tokens.
  • Risky AI-agent configurations are not limited to software engineers, as Mitiga manually verified findings tied to roles including founders, CEOs, product managers, researchers, and PhD students.
  • A benign dependency can still create a supply-chain execution risk when an agent is granted standing permission to fetch and execute content from an external URL controlled by someone else.

EXEC Executive Overview

This digest covers a focused but significant development in the emerging security landscape around AI agents and developer-assistant ecosystems. Mitiga Labs reported that AI instruction files and agent configuration artifacts are increasingly showing malware-like tradecraft, including prompt exfiltration, model endpoint hijacking, permission bypasses, and exposed credentials.

01 AI Agent Instruction Files as a Malware and Supply-Chain Surface

Mitiga Labs published research based on a six-month investigation into AI agent artifacts such as Cursor rules, Anthropic Skills, Claude Hooks, AGENTS.md and CLAUDE.md files, MCP server configurations, package.json droppers, and hidden .pyc bytecode files. The findings indicate that files intended to guide or configure AI agents can also carry techniques associated with spyware, credential exposure, execution abuse, and supply-chain manipulation.

The research identified prompt-exfiltration tradecraft, attacker-controlled ANTHROPIC_BASE_URL overrides, and permission-bypass defaults embedded in AI instruction and configuration files. The ANTHROPIC_BASE_URL behavior is particularly notable because it can redirect model traffic toward attacker-controlled infrastructure when agent environments honor the variable, creating an opportunity for interception or manipulation of AI workflows.

Mitiga also reported finding more than 1,230 hardcoded API keys and JWT tokens across the examined public repository content. The company said it manually verified in-the-wild findings and did not access or authenticate with the discovered credentials, framing the exposure as part of a broader pattern of sensitive data and control logic being embedded in artifacts consumed by AI coding assistants and agentic systems.

02 Skillgate Scanner and Detection Coverage

Alongside the research, Mitiga Labs released Skillgate, a free scanner designed to detect risky patterns across AI instruction files, skills, hooks, agent brain files, MCP configurations, supply-chain droppers, and bytecode artifacts. According to Mitiga, Skillgate has scanned more than 50,000 files across over 7,000 public repositories.

The tool currently includes more than 80 detection rules organized into technique families including direct execution, prompt manipulation, MCP server poisoning, supply-chain attacks, obfuscation, and credential exposure. This rule structure reflects how AI agent ecosystems are beginning to develop their own recognizable abuse patterns, overlapping with traditional malware and software supply-chain techniques while adding AI-specific vectors such as prompt manipulation and agent instruction poisoning.

The reporting highlights a shift in attacker opportunity from conventional source code alone to the broader operational context surrounding AI-assisted development. As AI agents increasingly ingest repository-level instructions and configuration files, those artifacts become part of the executable or decision-influencing environment, expanding the set of files that may affect developer workflows, automation behavior, and sensitive credential handling.
