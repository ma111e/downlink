Cybersecurity researchers at Xint.io and Theori have publicly disclosed details regarding CVE-2026-31431, a critical Linux local privilege escalation vulnerability. Tracked with a CVSS score of 7.8, the flaw is codenamed Copy Fail and enables any unprivileged local user to escalate privileges to root. The vulnerability stems from a logic flaw within the Linux kernel’s algif_aead cryptographic module, which was introduced by a source code commit in August 2017. Exploitation is achieved through a straightforward 732-byte Python script that executes in four distinct phases. The script first opens an AF_ALG socket and binds it to authencesn(hmac(sha256),cbc(aes)). It then constructs a shellcode payload and triggers a write operation targeting the kernel’s cached copy of a setuid binary such as /usr/bin/su. Finally, the script calls execve to load the injected shellcode and execute it with root privileges. Although the flaw cannot be exploited remotely, local attackers can reliably corrupt the page cache of unowned files to achieve code execution. The vulnerability also carries significant cross-container implications because the page cache is shared across all processes on the system. In response to the disclosure, major Linux distributions including Amazon Linux, Debian, Red Hat Enterprise Linux, SUSE, and Ubuntu have published official security advisories. Security experts note that Copy Fail echoes the Dirty Pipe vulnerability, sharing the same class of page cache corruption primitive but operating within a different subsystem. Bugcrowd researcher David Brumley highlighted that the 2017 in-place optimization incorrectly routes page-cache pages into a writable scatterlist for AEAD operations. This architectural quirk allows an unprivileged process to drive splice() into the socket and complete targeted writes into the page cache. Unlike many kernel exploits, Copy Fail does not require race conditions or kernel offsets to function reliably. The researchers emphasize that the vulnerability is uniquely portable, tiny, stealthy, and capable of bypassing container sandboxing across all affected Linux versions.

Vulnerability Overview

Researchers from Xint.io and Theori have disclosed CVE-2026-31431, a high-severity Linux local privilege escalation (LPE) flaw codenamed Copy Fail. The vulnerability carries a CVSS score of 7.8 and fundamentally alters how unprivileged users interact with the kernel’s page cache. By exploiting a logic flaw in the kernel’s cryptographic subsystem, attackers can write four controlled bytes into the page cache of any readable file on the system. This primitive ultimately grants the attacker full root access without requiring remote network access.

Technical Mechanism and Exploitation

The core issue originates from a logic flaw within the algif_aead module, introduced via a source code commit in August 2017. Exploitation is achieved through a compact 732-byte Python script that follows a precise four-step sequence. The script first opens an AF_ALG socket and binds it to authencesn(hmac(sha256),cbc(aes)). It then constructs a shellcode payload within the script environment. Next, the script triggers a write operation that targets the kernel’s cached copy of a setuid binary, specifically /usr/bin/su. Finally, the script executes execve("/usr/bin/su"), which loads the injected shellcode and runs it with root privileges.

Impact and Cross-Container Scope

While the vulnerability is strictly local and cannot be exploited remotely, its impact extends far beyond a single user session. An unprivileged user can reliably corrupt the page cache of a setuid binary to achieve immediate root access. Because the Linux page cache is shared across all processes on the host, the vulnerability carries significant cross-container implications. This shared memory architecture allows a single local compromise to cascade across isolated containers, effectively neutralizing container sandboxing boundaries.

Vendor Response and Historical Context

Major Linux distributions have already responded to the disclosure by publishing official security advisories. Affected vendors include Amazon Linux, Debian, Red Hat Enterprise Linux, SUSE, and Ubuntu. Security analysts note that Copy Fail shares architectural similarities with the Dirty Pipe vulnerability (CVE-2022-0847), as both rely on page cache corruption primitives. However, Copy Fail operates within a different subsystem and leverages a distinct exploitation path. Bugcrowd researcher David Brumley clarified that the 2017 in-place optimization in algif_aead allows a page-cache page to incorrectly end up in the kernel’s writable destination scatterlist. This misconfiguration enables an unprivileged process to drive splice() into the socket and complete small, targeted writes into the page cache of files it does not own.

Key Differentiators and Reliability

The Copy Fail vulnerability stands out due to its consistent reliability and lack of complex exploitation requirements. Unlike many kernel flaws, it does not depend on race conditions or kernel offset detection to function. The researchers describe the vulnerability as uniquely portable, tiny, stealthy, and cross-container. These combined properties allow any user account, regardless of privilege level, to escalate to full administrative access. Furthermore, the exploit bypasses standard sandboxing mechanisms and remains effective across all Linux versions and distributions shipped since 2017.
