  • The Windows UEFI CA 2023 certificate is present in the device’s Secure Boot Signature Database, making the device eligible for the 2023-signed Windows Boot Manager to become the default.
  • The device is not already running the 2023-signed Windows Boot Manager.

Microsoft described the issue as involving certain Trusted Platform Module validation settings, including invalid PCR7 configurations. The problem was associated with updating boot files and the automatic switch to the 2023-signed Windows Boot Manager.

Affected systems

The article focuses on Windows Server 2025 systems affected after the April 2026 security update. Microsoft also said the issue may affect some Windows 11 systems.

Microsoft stated that the issue is unlikely to affect personal devices. The impacted configurations are typically found on enterprise systems managed by corporate IT teams.

Microsoft’s fix

Microsoft fixed the issue in the June 2026 cumulative updates:

  • KB5094125 for Windows Server 2025.
  • KB5093998 for Windows 11 23H2.

Microsoft said the update addresses an issue where some devices might enter BitLocker Recovery after updating boot files on systems with certain TPM validation settings. These settings include invalid PCR7 configurations.

How the fix works

Microsoft said devices with the incompatible Group Policy configuration are prevented from installing the 2023-signed Windows Boot Manager. This is intended to prevent the unexpected BitLocker recovery key prompt.

If a device is affected by this behavior, administrators may see Event ID 1032 in the System event log when installing Windows updates.

Guidance for administrators

Microsoft advised IT administrators who cannot yet deploy the June updates to remove the relevant Group Policy configuration before installing KB5082063 and later updates. Microsoft also advised ensuring that BitLocker bindings use the PCR7 profile.

For administrators who cannot remove the Group Policy before deployment, Microsoft said they can apply a Known Issue Rollback on affected devices. This prevents the automatic switch to the 2023 Boot Manager, which triggers the BitLocker recovery prompts.

Related prior BitLocker recovery issues

The article notes that Microsoft has addressed similar BitLocker recovery problems in previous Windows update cycles. In August 2024, Microsoft fixed a known issue that triggered BitLocker recovery prompts across all supported Windows versions after the July 2024 security updates.

In May 2025, Microsoft released emergency updates to address a similar issue that caused Windows 10 systems to enter BitLocker recovery after the May 2025 security updates.

  • The BitLocker recovery prompt is expected to appear only once on affected devices, and later restarts should not trigger it again if the Group Policy configuration is not changed.
