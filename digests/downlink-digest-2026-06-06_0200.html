Cisco said the issue could allow an authenticated local attacker to execute arbitrary commands as root by supplying a crafted file to an affected system. The company attributed the flaw to insufficient validation of user-supplied input, enabling command injection and privilege escalation.

Overview

Cisco warned that a high-severity security flaw affecting Cisco Catalyst SD-WAN Manager is being actively exploited. The vulnerability is tracked as CVE-2026-20245 and has a CVSS score of 7.8. Cisco Catalyst SD-WAN Manager was formerly known as SD-WAN vManage.

Cisco said there are currently no patches or mitigations available for CVE-2026-20245. The company recommended that customers upgrade their SD-WAN software to ensure they have applied fixes released for CVE-2026-20182 on May 14, 2026.

Affected deployment types

The article states that CVE-2026-20245 affects the following Cisco Catalyst SD-WAN Manager deployment types:

  • On-Prem Deployment
  • Cisco SD-WAN Cloud-Pro
  • Cisco SD-WAN Cloud (Cisco Managed)
  • Cisco SD-WAN for Government (FedRAMP)

Technical details

Cisco described CVE-2026-20245 as a vulnerability in the CLI of Cisco Catalyst SD-WAN Manager. The flaw could allow an authenticated local attacker to execute arbitrary commands as root by supplying a crafted file to an affected system.

Cisco said the vulnerability is caused by insufficient validation of user-supplied input. An attacker could exploit it by uploading a crafted file to the affected system. Successful exploitation could allow command injection and privilege escalation to the root user.

Exploit requirements

Cisco said an attacker must have netadmin privileges on the affected system to exploit CVE-2026-20245. The company stated that this would require valid credentials or exploitation of CVE-2026-20182 or CVE-2026-20127.

Cisco also said it is not aware of successful exploitation by other methods. This makes CVE-2026-20245 notable in the article because it can follow authentication-related compromises described elsewhere in the Cisco SD-WAN ecosystem.

Related vulnerabilities

The article links CVE-2026-20245 to two related Cisco SD-WAN vulnerabilities:

  • CVE-2026-20182, with a CVSS score of 10.0, was disclosed by Rapid7 and described as an authentication bypass that could allow unauthenticated remote attackers to obtain administrative privileges on susceptible systems.
  • CVE-2026-20127 is described as another authentication bypass affecting the same component and as similar to CVE-2026-20182.

Both CVE-2026-20182 and CVE-2026-20127 have been exploited in the wild as zero-days. The article says a threat activity cluster named UAT-8616 has been linked to abuse of CVE-2026-20127 as far back as 2023.

Observed impact

Cisco said it observed limited cases in which exploitation of CVE-2026-20245 resulted in a configuration change being pushed to edge devices. The article does not identify who is behind the latest exploitation efforts.

The vulnerability was discovered and reported by Google Mandiant researchers Chester Sng, Pete Boonyakarn, and Logeswaran Nadarajan, according to Cisco.

Detection guidance and risk warning

Cisco warned that internet-exposed systems are at heightened risk of compromise. To look for indicators of compromise, Cisco advised users to check the /var/log/scripts.log file.

The article gives examples of log entries involving script uploads, including:

  • Tenant list upload per vSmart serial number using /usr/bin/vconfd_script_upload_tenant_list.sh
  • vSmart upload serial numbers using /usr/bin/vconfd_script_upload_vsmart_serial_numbers.sh
  • ZTP upload chassis numbers using /usr/bin/vconfd_script_upload_chassis_number_file.sh

Broader Cisco SD-WAN exploitation context

The article states that CVE-2026-20245 is the seventh Cisco SD-WAN flaw to be flagged as actively exploited this year. The other cited flaws are:

  • CVE-2026-20182
  • CVE-2026-20127
  • CVE-2026-20122
  • CVE-2026-20128
  • CVE-2026-20133
  • CVE-2022-20775

The disclosure also follows Cisco addressing another high-severity flaw in Unified Communications Manager, CVE-2026-20230, which has a CVSS score of 8.6. Cisco said proof-of-concept exploit code is public for CVE-2026-20230, but there is no evidence that it has been actively exploited.

  • CVE-2026-20245 is especially concerning because Cisco says it can be chained after CVE-2026-20182 or CVE-2026-20127, turning prior authentication-bypass exposure into root-level command execution on affected systems.
  • The most actionable detection step Cisco provides is to inspect /var/log/scripts.log for suspicious script upload entries involving tenant lists, vSmart serial numbers, or chassis number files.
  • Internet-exposed Cisco Catalyst SD-WAN Manager systems face heightened risk, even though the flaw itself requires netadmin privileges, because related authentication-bypass vulnerabilities have already been exploited in the wild.
  • Cisco observed exploitation leading to configuration changes pushed to edge devices, indicating that compromise of SD-WAN Manager can affect downstream network infrastructure rather than remaining limited to the manager system.

EXEC Executive Overview

During this coverage window, reporting centered on active exploitation of a high-severity Cisco Catalyst SD-WAN Manager vulnerability with no patch currently available. The activity is notable because exploitation can lead to root-level command execution on affected management systems and, in limited observed cases, configuration changes propagated to SD-WAN edge devices.

01 Active Exploitation of Cisco Catalyst SD-WAN Manager

Cisco disclosed that CVE-2026-20245 in Cisco Catalyst SD-WAN Manager is being actively exploited. The flaw allows an authenticated local attacker with netadmin privileges to upload a crafted file and execute arbitrary commands as root on an affected system.

The exploitation path is constrained by credential and access requirements: Cisco stated that successful exploitation requires valid credentials or prior exploitation of related vulnerabilities, specifically CVE-2026-20182 or CVE-2026-20127. This places CVE-2026-20245 within a broader intrusion chain in which attackers may first obtain access through other flaws or compromised accounts before escalating control over the SD-WAN management plane.

Cisco reported limited cases in which exploitation of CVE-2026-20245 resulted in a configuration change being pushed to edge devices. That detail is significant for defenders because SD-WAN Manager occupies a central orchestration role, and unauthorized changes at the management layer can have downstream operational impact across distributed network infrastructure.

02 Patch Status and Exposure Context

At the time of reporting, Cisco said there were no patches or mitigations available for CVE-2026-20245. The company linked exposure to the broader vulnerability set affecting SD-WAN software and referenced fixes for CVE-2026-20182 as part of the surrounding risk context.

The reported exploitation remains described as limited, but the vulnerability’s location in a network management platform increases its strategic importance. For threat hunters, detection engineers, SOC analysts, and incident responders, the key intelligence value is the combination of active exploitation, root-level execution potential, authenticated access requirements, and observed propagation of configuration changes to managed edge infrastructure.
