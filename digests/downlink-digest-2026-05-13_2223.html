This article details a critical unauthenticated remote code execution vulnerability in Exim, a widely deployed default mail server on Linux distributions. The flaw enables full server compromise and potential network pivoting, requiring immediate remediation via the newly released version 4.99.3.
A critical use-after-free vulnerability in Exim versions 4.97 through 4.99.2 with GnuTLS enables unauthenticated remote code execution, making immediate updates to version 4.99.3 essential. The flaw was successfully weaponized using a combination of autonomous AI tools and human research, underscoring the urgent need for rapid patching across widely deployed Linux mail servers.

A critical vulnerability in certain Exim mail server configurations allows unauthenticated attackers to execute arbitrary code.

Identified as CVE-2026-45185, the flaw impacts Exim versions 4.97 through 4.99.2 when compiled with GnuTLS and configured to advertise STARTTLS and CHUNKING.

The issue is a use-after-free condition triggered during TLS shutdown while processing BDAT chunked SMTP traffic.

When the server frees a TLS transfer buffer, it continues to use stale callback references that write data into the freed memory region.

This memory corruption can lead to remote code execution on the affected server.

The vulnerability was discovered and reported by XBOW researcher Federico Kirschbaum on May 1.

A patched version, Exim 4.99.3, was released on May 12 to address the issue.

Administrators using Ubuntu or Debian-based systems should immediately apply the update through their package managers.

Notably, Exim builds compiled with OpenSSL are completely unaffected by this vulnerability.
