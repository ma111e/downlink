Overview

Hackers are actively exploiting a critical vulnerability tracked as CVE-2026-3300 in the Everest Forms Pro plugin for WordPress. The flaw affects Everest Forms Pro versions 1.9.12 and earlier. According to the article, the vulnerability can be exploited without authentication to execute arbitrary code on the server.

Everest Forms Pro is described as a commercial add-on for the WordPress form builder plugin Everest Forms. The plugin is used to create contact, registration, payment, and other custom application forms. The article says exploitation of the flaw can allow attackers to take complete control of a WordPress website.

Technical Details

The vulnerability is in the plugin’s Complex Calculation feature. The article explains that this feature accepts values submitted through form fields and inserts them into a PHP code string. The resulting code is then executed using PHP’s eval() function.

Although submitted input is passed through sanitize_text_field(), the article says this function does not escape single quotes or other characters that influence PHP syntax. As a result, an attacker can close the intended string, inject arbitrary PHP code, and comment out the remaining generated code. This lets the attacker achieve code execution on the server.

Wordfence describes an observed exploitation method in which the attacker submits a value for a text field that begins with a single quote. That quote closes the wrapping string literal. The attacker then includes a PHP statement that calls wp_insert_user() to create a new administrator account.

The article says the observed malicious administrator username is “diksimarina.” Wordfence explains that a trailing // comment marker ensures the rest of the generated PHP code, including the closing quote, is treated as a comment and does not cause a syntax error. When the form is processed and the calculation is evaluated, the injected PHP code is executed and the malicious administrator account is created.

Impact

The article says administrator-level access gives attackers full power to perform high-risk actions on a breached website. These actions include:

  • Modifying website content.
  • Installing plugins and themes.
  • Planting backdoors and webshells.
  • Accessing private databases.

Wordfence telemetry shows the vulnerability is being exploited in the wild to create rogue administrator accounts. The article frames this as a path to complete takeover of affected WordPress websites.

Discovery, Patch, and Exploitation Timeline

Researcher h0xilo submitted CVE-2026-3300 through Wordfence in February. The Everest Forms developer released a patch addressing the issue on March 18.

According to Wordfence data cited in the article, active exploitation started on April 13. Wordfence says its firewall has blocked more than 29,300 exploitation attempts.

Indicators and Defensive Recommendations

  • 209.146.60.26

The article says Wordfence recommends that defenders block those IP addresses. It also notes that Wordfence’s report provides several offending IP addresses as indicators of compromise.

Website administrators are recommended to review log files and administrator accounts for suspicious activity. The article specifically calls out the string “diksimarina” as something administrators should look for during review.
