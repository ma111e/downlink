Most severe remote code execution issues

A leading vulnerability in the release is CVE-2026-45657, a Windows Kernel use-after-free flaw with a CVSS score of 9.8. Microsoft said exploitation could occur through specially crafted network traffic sent to a vulnerable Windows system.

According to Microsoft, malicious network packets could trigger a flaw in how the Windows kernel processes certain TCP/IP data. Successful exploitation could allow an attacker to run code with system-level privileges without signing in or requiring user interaction.

The article also highlights two other critical remote code execution vulnerabilities:

  • CVE-2026-47291, a Windows HTTP.sys integer overflow or wraparound flaw rated 9.8 that allows an unauthorized attacker to execute code over a network.
  • CVE-2026-44815, a Windows DHCP Client stack-based buffer overflow vulnerability rated 9.8 that allows an unauthorized attacker to execute code over a network.

Alex Vovk of Action1 said CVE-2026-44815 requires no credentials or user action and can turn network traffic into a full system compromise. He said an attacker could send specially crafted network traffic to a system configured for DHCP services, and that successful exploitation could allow unauthorized code execution over the network with high impact to confidentiality, integrity, and availability.

The article states that DHCP is a core network function and that successful exploitation of CVE-2026-44815 could lead to server compromise, malware deployment, data theft, service disruption, and deeper movement into the network. Systems handling DHCP traffic are described as high-priority patch targets.
