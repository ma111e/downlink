INFINITERED Malware Capabilities

INFINITERED trojanized REDCap’s own system files and performed three main functions:

  • It hijacked the REDCap upgrade process so that each new REDCap version reinjected the malicious code instead of removing it.
  • It harvested usernames and passwords from the login page and stored them, encrypted, in local database tables.
  • It acted as a backdoor by accepting commands through HTTP cookies and running on every page load.

After establishing access on the REDCap server, UNC6508 performed internal reconnaissance and credential discovery. The group collected database and service account credentials, used those logins to move into the internal network, and ultimately reached a domain administrator account. Google did not describe the exact path used to obtain the administrator account.

Email Exfiltration Through Google Workspace Rules

The most notable part of the campaign was the email exfiltration method. Instead of using malware on the mail server, a separate exfiltration tool, or traffic that would appear unusual on the network, UNC6508 abused a legitimate Google Workspace administrative feature.

The group used content compliance rules, which can scan mail for keywords and copy or forward matching messages. The attackers created a rule misspelled “Patroit” that watched for nearly 150 keywords, search terms, and email addresses. When a message matched the rule, Google Workspace silently BCC’d it to an attacker-controlled Gmail address, which Google later disabled.

The article says the same unchecked write appears on other key-management endpoints. It also says the fix required three pull requests to land.

Once the route gate is bypassed, administrative handlers become reachable. CVE-2026-47102 is a privilege escalation issue in the /user/update endpoint.

Varonis characterizes this as a race condition because sanitization is applied as a post-processing step, while the browser renders streamed content incrementally.

When Bing receives the request, its backend performs a server-side fetch of the supplied image URL. Because this fetch is made from Bing’s infrastructure rather than the victim’s browser, the browser’s CSP does not apply. The attacker’s server can then log the URL path containing the exfiltrated data.
