Malware Behavior

NFCShare displays fake verification screens that prompt victims to place payment cards near the mobile device’s NFC chip. The malware then reads card data using Android’s IsoDep interface and EMV commands.

The data stolen includes:

  • Card number
  • Card type
  • Expiry date
  • A 4-digit PIN entered by the victim under the pretense of a security step

The malware exfiltrates the collected information to the attacker’s command-and-control host over a WebSocket channel.

A notable change in the newer NFCShare version is malformed APK packaging intended to hinder automated analysis and potentially security tools. The APK remains a ZIP archive, but newer samples include poisoned or malformed file paths inside the ZIP.

These malformed paths can cause some extraction tools to incorrectly interpret internal relative paths as filesystem paths and trigger errors. D3Lab notes that this technique does not prevent manual analysis or code recovery, but can disrupt static analysis in some tools.
