Overview

Hackers are targeting WordPress websites that run vulnerable versions of the WP Maps Pro plugin. The issue allows attackers to create rogue administrator accounts without authentication, giving them full administrative access to affected sites.

The vulnerability is tracked as CVE-2026-8732 and is described in the article as having a critical severity rating. It affects WP Maps Pro versions 6.1.0 and older. WP Maps Pro is a premium plugin for creating interactive and customizable maps and store locators, with support for providers such as Google Maps and OpenStreetMap.

Plugin Context and Exposure

The article says WP Maps Pro is commonly used by:

  • Businesses
  • Real estate websites
  • Travel sites
  • Directories
  • Organizations displaying multiple locations on a map

It also notes that the plugin has over 15,800 sales on the Envato Market. That gives a sense of the potential footprint of the vulnerable software.

Discovery and Reporting Timeline

According to the article:

  • Security researcher David Brown discovered and reported the flaw.
  • Brown reported it to Wordfence on March 24.
  • After validating the exploit, the vendor was notified on May 16.
  • WP Maps Pro 6.1.1 was released on May 20 with a fix.

The article recommends that website administrators update their plugin as soon as possible because malicious activity has already been observed.

Technical Details of the Vulnerability

The flaw is caused by a “temporary access” feature in the plugin. This feature was intended to allow vendor support staff to access customer websites for troubleshooting.

Brown found that the AJAX endpoint for this feature was accessible to unauthenticated users. The endpoint relied solely on a nonce check that was publicly exposed in frontend JavaScript, which the article says rendered the protection ineffective.

The article explains that a specially crafted request can trigger code that:

  1. Creates a new WordPress user.
  2. Assigns that user the administrator role.
  3. Generates a passwordless login URL.
  4. Sends that URL to a remote system.

Wordfence researchers further explain that when the request is made with a check_temp parameter set to false, the function calls wp_insert_user() to create a new user with:

  • A hardcoded administrator role
  • A randomly generated username
  • The hardcoded email address support@flippercode.com

The function then generates a “magic login URL” using generate_login_link(), stores it as user meta, and returns it in the response body.

Exploitation Flow and Access Gained

The article states that once the attacker visits the generated login URL, they are automatically authenticated to the newly created administrator account. No password or other verification is required.

This makes the vulnerability especially dangerous because it does not merely create an account; it also provides a direct way to log in to that account immediately.

Active Exploitation

Researchers at Defiant, the company behind Wordfence, observed threat actors attempting to exploit the vulnerability. The article says they blocked more than 3,600 attempts over a 24-hour period.

This indicates that the vulnerability is already being targeted in real-world attacks.

Potential Impact

The article highlights that administrator-level access can enable attackers to:

  • Inject persistent backdoors
  • Modify website content
  • Access private data
  • Deploy web shells
  • Install malicious plugins
  • Take over the website

Because the exploit grants administrator privileges, the resulting impact can extend across the entire WordPress instance.

Vendor Response and Recommended Action

WP Maps Pro version 6.1.1 was released on May 20 to fix CVE-2026-8732. The article advises website administrators to update immediately.

Given that malicious activity has already been observed, the article frames patching as urgent rather than precautionary.

EXEC Executive Overview

The reporting in this coverage window highlights two sharply different but operationally important security themes: rapid exploitation of exposed internet applications and large-scale disruption of criminal infrastructure. Together, the stories show both the speed at which newly disclosed web flaws can be weaponized and the scale of device abuse underpinning proxy-enabled cybercrime ecosystems.

The most urgent defensive signal is active exploitation of a critical WordPress plugin vulnerability that enables full administrator compromise. In parallel, Dutch authorities announced a major botnet disruption tied to a proxy service reportedly backed by millions of infected devices across consumer, mobile, and IoT environments.

01 Active Exploitation of Public-Facing Web Applications

Attackers are actively targeting WordPress sites running vulnerable versions of WP Maps Pro through CVE-2026-8732, a critical flaw affecting version 6.1.0 and earlier. The issue stems from a temporary access feature whose AJAX endpoint was reachable without authentication and depended on a nonce exposed in frontend JavaScript, undermining its intended access control.

According to reporting from Wordfence, successful exploitation allows an attacker to create a new WordPress administrator account, generate a passwordless login URL, and automatically authenticate into the newly created admin session. That chain effectively turns a plugin-level bug into immediate full-site administrative takeover, making it especially relevant for threat hunters monitoring suspicious account creation, privilege escalation, and anomalous WordPress admin activity.

Wordfence said it blocked more than 3,600 exploitation attempts in a 24-hour period, indicating broad and active abuse rather than isolated proof-of-concept scanning. The vendor released WP Maps Pro 6.1.1 on May 20 to address the flaw, but the reporting suggests a meaningful installed base remained exposed long enough to attract opportunistic attacker activity.

02 Botnet and Proxy Infrastructure Disruption

Dutch authorities announced the dismantling of a botnet linked to at least 17 million infected devices, spanning computers, smartphones, tablets, and IoT systems. The Dutch Politie and National Cyber Security Center said more than 200 servers in the Netherlands functioned as backend infrastructure supporting the operation.

Police reportedly seized part of the server estate from a hosting provider, after which the provider took the broader botnet infrastructure offline once its criminal use was identified. Local reporting cited by The Hacker News identified the disrupted service as Asocks, a company known for offering residential, mobile, and corporate proxy services.

The case underscores the continuing overlap between botnet operations and commercialized proxy ecosystems, where compromised endpoints can be monetized as relay infrastructure. For defenders, the significance lies less in a single malware family and more in the persistence of large-scale device hijacking as a foundational service layer for anonymization, fraud enablement, and follow-on intrusion activity.
