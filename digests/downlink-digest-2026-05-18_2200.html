Top Attacks and Breaches

Vodafone source code leak

Vodafone, described as a major international telecom, sustained a source code leak claimed by the Lapsus$ extortion group. The company confirmed limited access to GitHub files through compromised third-party development software. Vodafone stated that customer data and core network infrastructure were not affected by the incident.

THORChain cryptocurrency theft

THORChain, a Switzerland-based cryptocurrency platform, encountered a security breach that led to the theft of about $10.7M. Trading was halted after one of six vaults was compromised. The company said losses were limited to protocol-owned assets across several blockchains.

West Pharmaceutical Services ransomware attack

West Pharmaceutical Services, a global manufacturer of drug delivery components, experienced a ransomware attack. The incident disrupted shipping, manufacturing, and shared service functions. The company disclosed that some systems were encrypted and data was stolen, but no ransomware group had publicly claimed responsibility.

Foxconn cyberattack

Foxconn, a global electronics manufacturer, confirmed a cyberattack on its North American operations. The confirmation followed a claim by the Nitrogen ransomware group that it had stolen 8TB of data. Foxconn confirmed disruption at some factories and said affected facilities were resuming normal production.

AI Threats

OpenClaw “Claw Chain” vulnerabilities

Researchers unveiled “Claw Chain,” a set of four vulnerabilities in OpenClaw, an autonomous AI agent platform. The flaws allow attackers to bypass sandbox controls, expose restricted files, leak secrets, and gain owner-level access. One of the flaws is CVE-2026-44112, described as critical and rated CVSS 9.6.

AI-assisted macOS kernel exploit

Researchers developed an AI-assisted macOS kernel exploit that bypasses Apple’s Memory Integrity Enforcement on M5 chips. The exploit grants full system control on macOS 26.4.1. Anthropic’s Mythos Preview reportedly accelerated bug discovery, and the findings were privately reported to Apple before public disclosure.

Vercel v0.dev abuse for phishing

Researchers detailed how threat actors abuse Vercel’s AI website generator, v0.dev, to mass-produce realistic phishing pages. The phishing pages mimic brands such as Microsoft and Spotify. The campaigns use Telegram bots to capture credentials and payment details in real time.

Malware in Hugging Face repository

Researchers found a popular Hugging Face repository hiding Windows-targeting malware after it amassed more than 200,000 downloads. The package posed as OpenAI’s privacy filter. It installed an infostealer that harvested browser passwords, cookies, SSH keys, VPN configurations, and cryptocurrency wallets before exfiltrating the data.

Vulnerabilities and Patches

Windows YellowKey and GreenPlasma zero-days

Two Windows zero-day vulnerabilities, YellowKey and GreenPlasma, affect Windows 11 and recent Windows Server versions. YellowKey allows BitLocker bypass through Windows Recovery Environment with physical access. GreenPlasma abuses the CTFMON framework to escalate privileges to SYSTEM.

Proof-of-concept code for these vulnerabilities is public. The vulnerabilities are still unpatched, according to the article.

NGINX CVE-2026-42945

F5 fixed CVE-2026-42945, a critical memory flaw in the NGINX rewrite module. The flaw affects NGINX versions 0.6.27 through 1.30.0 and is described as an 18-year-old bug. It enables denial of service and, under specific configurations, possible remote code execution.

The article states that public exploit code requires memory protections to be disabled. It also notes that Check Point IPS provides protection against Nginx Heap Overflow identified as CVE-2026-42945.

Cisco Catalyst SD-WAN CVE-2026-20182

Cisco addressed CVE-2026-20182, a critical authentication bypass in Catalyst SD-WAN controllers. The flaw is being actively exploited and allows remote, unauthenticated attackers to gain full administrative control of affected systems. CISA ordered federal agencies to patch vulnerable devices following Cisco’s fixes.

Apple CVE-2026-28819

Apple released security updates for CVE-2026-28819. The vulnerability is an out-of-bounds write flaw in the Wi-Fi component affecting iOS, iPadOS, and macOS. Successful exploitation could allow an app to execute code with kernel privileges, and Apple addressed the issue with improved bounds checking.

The report also details the use of Fortinet and Cisco vulnerabilities, NTLM relay, and OWA/M365 for initial access in attacks. The article states that Check Point Threat Emulation and Harmony Endpoint provide protection against this threat.

Qilin led with 338 victims. The Gentlemen rose to third place, and LockBit 5.0 returned with 163 victims.

World Cup 2026 cyber activity

Check Point Research quantified a World Cup 2026-driven surge in cyber activity. Weekly attacks per organization rose in Mexico, Canada, and the United States in April. The affected sectors included media, hospitality, transportation, and travel.

FIFA-themed domains reached 9,741 in April. By early May, one in 41 of those domains were malicious.

FamousSparrow intrusion against Azerbaijani oil and gas company

Researchers attributed a months-long intrusion against an Azerbaijani oil and gas company to the Chinese-linked FamousSparrow group. The attackers exploited an unpatched Microsoft Exchange server to deploy web shells. They then alternated between Deed RAT and TernDoor across three waves of persistent activity.

Ivanti Xtraction

Ivanti fixed a critical vulnerability in Ivanti Xtraction tracked as CVE-2026-8043. The flaw has a CVSS score of 9.6 and affects Ivanti Xtraction before version 2026.2.

Ivanti described the issue as external control of a file name. According to the advisory cited in the article, a remote authenticated attacker could exploit the flaw to read sensitive files and write arbitrary HTML files to a web directory. The stated impact includes information disclosure and possible client-side attacks.

Fortinet FortiAuthenticator and FortiSandbox

Fortinet published advisories for two critical vulnerabilities affecting FortiAuthenticator and FortiSandbox-related products. Both flaws have CVSS scores of 9.1 and may allow unauthenticated attackers to execute unauthorized code or commands.

The first vulnerability is CVE-2026-44277, an improper access control issue in FortiAuthenticator. The article says it may allow an unauthenticated attacker to execute unauthorized code or commands via crafted requests. Fortinet fixed the issue in FortiAuthenticator versions 6.5.7, 6.6.9, and 8.0.3.

The second vulnerability is CVE-2026-26083, a missing authorization vulnerability in the FortiSandbox, FortiSandbox Cloud, and FortiSandbox PaaS WEB UI. The article says it may allow an unauthenticated attacker to execute unauthorized code or commands via HTTP requests. It was fixed in FortiSandbox versions 4.4.9 and 5.0.2, FortiSandbox Cloud version 5.0.6, and FortiSandbox PaaS versions 4.4.9 and 5.0.2.

SAP S/4HANA and SAP Commerce Cloud Configuration

SAP shipped fixes for two critical vulnerabilities, both with CVSS scores of 9.6. These are CVE-2026-34260 in SAP S/4HANA and CVE-2026-34263 in SAP Commerce cloud configuration.

CVE-2026-34260 is an SQL injection vulnerability in SAP S/4HANA. The article says it could be exploited by an attacker to inject malicious SQL statements and potentially impact the confidentiality and availability of the application. It also states that because the affected code only allows read access to data, the vulnerability does not compromise application integrity.

Pathlock said CVE-2026-34260 allows a low-privileged, authenticated attacker to inject malicious SQL code through user-controlled input. According to Pathlock, the issue could expose sensitive database information and crash the application.

CVE-2026-34263 is a missing authentication check in SAP Commerce cloud configuration. Onapsis said the vulnerability is caused by an overly permissive security configuration with improper rule ordering. According to Onapsis, it allows an unauthenticated user to perform malicious configuration upload and code injection, resulting in arbitrary server-side code execution.

VMware Fusion

Broadcom released patches for CVE-2026-41702 in VMware Fusion. The flaw is rated high severity with a CVSS score of 7.8 and was addressed in VMware Fusion version 26H1.

Broadcom described the issue as a TOCTOU, or Time-of-check Time-of-use, vulnerability. The vulnerability occurs during an operation performed by a SETUID binary. According to Broadcom, a malicious actor with local non-administrative user privileges may exploit the flaw to escalate privileges to root on a system where Fusion is installed.

n8n Critical Vulnerabilities

The article lists five critical vulnerabilities affecting n8n. Each has a CVSS score of 9.4, and the impacts include remote code execution, arbitrary file reading, and full compromise of the n8n server or host.

The first issue is CVE-2026-42231. It affects the xml2js library used to parse XML request bodies in n8n’s webhook handler. A crafted XML payload can allow prototype pollution, enabling an authenticated user with permission to create or modify workflows to achieve remote code execution on the n8n host. The issue was fixed in n8n versions 1.123.32, 2.17.4, and 2.18.1.

The second issue is CVE-2026-42232. It allows an authenticated user with permission to create or modify workflows to achieve global prototype pollution via the XML Node. The article says this can lead to remote code execution when combined with other nodes exploiting the prototype pollution. It was fixed in n8n versions 1.123.32, 2.17.4, and 2.18.1.

The third issue is CVE-2026-44791. The article describes it as a bypass for CVE-2026-42232 that could result in remote code execution on the n8n host. It was fixed in n8n versions 1.123.43, 2.20.7, and 2.22.1.

The fourth issue is CVE-2026-44789. It allows an authenticated user with permission to create or modify workflows to achieve global prototype pollution through an unvalidated pagination parameter in the HTTP Request node. The article says this can lead to remote code execution on the n8n host. It was fixed in n8n versions 1.123.43, 2.20.7, and 2.22.1.

The fifth issue is CVE-2026-44790. It allows an authenticated user with permission to create or modify workflows to inject CLI flags on the Git node’s Push operation. According to the article, this enables an attacker to read arbitrary files from the n8n server and results in full compromise. It was fixed in n8n versions 1.123.43, 2.20.7, and 2.22.1.
