Attackers are running an active malvertising campaign that abuses Google Ads and legitimate Claude.ai shared chats to distribute macOS malware. Users searching for Claude downloads encounter sponsored results that appear to link to the official claude.ai domain but actually direct them to weaponized shared chat pages. These chats mimic official installation guides and trick users into pasting base64 commands into their Terminal. The commands silently download and execute shell scripts that run entirely in memory. One variant performs regional profiling by checking for Russian or CIS keyboard layouts before delivering a payload via macOS scripting tools. A second variant bypasses profiling and immediately harvests browser credentials, cookies, and Keychain contents using the MacSync infostealer. The campaign uniquely leverages legitimate domains to bypass traditional phishing detection methods. Security researchers recommend downloading AI tools only from official sources and avoiding pasting unverified terminal commands.

A new malvertising campaign is actively targeting macOS users by abusing both Google Ads and legitimate Claude.ai shared chat features. The attack was initially identified by Trendyol Group security engineer Berk Albayrak, who documented the campaign on LinkedIn. Users searching for Claude downloads encounter sponsored search results that list claude.ai as the destination but lead to malicious shared chats. These chats impersonate official installation guides and are falsely attributed to Apple Support. The pages instruct victims to open Terminal and paste base64-encoded commands that silently download shell scripts from external domains. Researchers discovered two distinct variants of the attack that follow identical social engineering structures but utilize different infrastructure and payloads. The first variant checks the victim’s system for Russian or CIS-region keyboard layouts and exits silently if a match is found. Machines that pass this filter receive a profiling step that collects the external IP address, hostname, OS version, and locale before sending the data to the attacker. The script then executes a second-stage payload using macOS’s built-in osascript engine, achieving remote code execution without dropping traditional binaries on disk. The second variant skips the profiling phase entirely and immediately harvests browser credentials, cookies, and macOS Keychain contents. This data is packaged and exfiltrated to the attacker’s server, identifying the malware as a variant of the MacSync infostealer. Unlike traditional malvertising that relies on lookalike domains, this campaign points directly to Anthropic’s genuine claude.ai domain. The malicious instructions are hosted natively within Claude’s shared chat feature, rendering standard URL verification ineffective. Similar abuse of AI platform chats has been previously observed targeting ChatGPT and Grok users. Users are advised to navigate directly to claude.ai for official downloads and to treat any terminal commands from chat interfaces with extreme caution.

Campaign Overview and Delivery Mechanism

Attackers are executing an active malvertising campaign that weaponizes both Google Ads and legitimate Claude.ai shared chats to distribute macOS malware. The delivery method flips traditional malvertising tactics by pointing sponsored search results directly to Anthropic’s genuine claude.ai domain rather than using lookalike phishing sites. Users searching for Claude downloads encounter these ads, which lead to malicious shared chat pages disguised as official installation guides. These chats are falsely attributed to Apple Support and employ a consistent social engineering approach across multiple variants.

Technical Execution and Payload Analysis

The malicious chats instruct victims to open Terminal and paste base64-encoded commands that silently download shell scripts from external domains. Researchers identified two distinct variants utilizing different infrastructure but following identical structural patterns.

  • The first variant, documented by security engineer Berk Albayrak, retrieves an encoded script that functions as a MacSync infostealer variant. It immediately harvests browser credentials, cookies, and macOS Keychain contents before packaging and exfiltrating the data to the attacker’s server.
  • The second variant, identified by BleepingComputer, incorporates a geographic filtering mechanism. It checks for Russian or CIS-region keyboard layouts and exits silently with a cis_blocked status ping if a match is detected. Only machines that pass this check proceed to a profiling step that collects the external IP address, hostname, OS version, and keyboard locale. This data is sent to the attacker before the script executes a second-stage payload via macOS’s built-in osascript engine. This method achieves remote code execution while running entirely in memory and leaving no traditional binary or application dropped on disk.

Researcher Findings and Historical Context

The campaign was first spotted by Berk Albayrak, a security engineer at Trendyol Group, who publicly shared his findings on LinkedIn. BleepingComputer researchers verified Albayrak’s initial discovery and located a second shared Claude chat conducting the same attack through entirely separate infrastructure. Both chats remained publicly accessible at the time of reporting. This technique mirrors previous campaigns that abused AI platform shared chats to target ChatGPT and Grok users. The use of legitimate domains and native chat features represents an evolution in malvertising that bypasses conventional domain-based threat detection.

Mitigation Recommendations

Users are strongly advised to navigate directly to claude.ai for downloading the native Claude app rather than clicking sponsored search results. The legitimate Claude Code CLI is available through official documentation and does not require pasting commands from a chat interface. Security best practices dictate treating any instructions requesting terminal command pasting with extreme caution, regardless of the perceived source. BleepingComputer contacted both Anthropic and Google for comment prior to publishing the findings.
