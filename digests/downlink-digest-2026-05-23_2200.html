A critical SQL injection vulnerability tracked as CVE-2026-26980 affects Ghost CMS from version 3.24.0 up to 6.19.0. The flaw allows unauthenticated attackers to read arbitrary information from the database, most critically the admin API keys that grant full control over users, articles, and themes. The Ghost project released a patch in version 6.19.1 on February 19, 2026, but many site owners failed to apply the update promptly.

Threat intelligence researchers at XLab (Qianxin) observed wide-scale exploitation of this vulnerability as part of a ClickFix campaign. More than 700 domains were confirmed compromised, including Harvard University, Oxford University, Auburn University, and the privacy-focused search engine DuckDuckGo. The breadth of targets—spanning education, AI/SaaS, media, fintech, and security sites—highlights the opportunity created by slow patching.

The attack chain begins with the SQL injection to exfiltrate admin API keys. Armed with these elevated privileges, the attackers authenticate and inject a lightweight JavaScript loader into published articles. This loader retrieves a second-stage cloaking script from attacker-controlled infrastructure. The cloaking module fingerprints each visitor and, when satisfied, overlays a fake Cloudflare browser check on the article page via an iframe.

The fake prompt is a classic ClickFix lure: it displays a message telling the user to prove they are human by pasting a supplied command into the Windows Command Prompt. Executing the command drops the final payload onto the victim’s system. Multiple payloads have been observed, including DLL loaders, JavaScript droppers, and an Electron-based malware binary named UtilifySetup.exe. The threat landscape is further complicated by the presence of at least two distinct activity clusters—one sometimes cleans another’s injected script to deploy its own, and SentinelOne also documented separate clusters.

To remediate, Ghost CMS administrators should upgrade immediately to version 6.19.1 or later and rotate any previously used admin API keys, as they may have already been exposed. A thorough review of all article content for malicious scripts is essential, aided by the indicators of compromise shared by XLab. Additionally, the researchers advise keeping a 30-day record of admin API call logs to support retrospective investigation should a breach be suspected.

Comprehensive Analysis: Ghost CMS SQL Injection Exploitation in Large-Scale ClickFix Campaign

Vulnerability Overview

A critical SQL injection vulnerability (CVE-2026-26980) exists in Ghost CMS versions 3.24.0 through 6.19.0. The flaw permits unauthenticated attackers to read arbitrary data from the website’s database. Most significantly, attackers can extract the admin API keys that provide full management access to users, articles, and themes. With these keys, an attacker can modify article content and inject JavaScript.

Patch and Disclosure Timeline

  • February 19, 2026: The Ghost project released version 6.19.1, which fixes the vulnerability, along with the security advisory GHSA-w52v-v783-gw97.
  • February 27, 2026: SentinelOne published a vulnerability database entry detailing active exploitation of CVE-2026-26980 and methods for detecting related incidents.
  • Subsequent weeks: Despite the availability of the patch, many vulnerable sites remained unpatched, allowing large-scale exploitation.

Campaign Discovery and Scope

Researchers at XLab (the threat intelligence arm of the Chinese cybersecurity firm Qianxin) uncovered a wide-ranging campaign that leverages CVE-2026-26980 to inject ClickFix malware delivery flows into Ghost CMS sites. The campaign confirmed impact on:

  • Over 700 domains.
  • Sites belonging to Harvard University, Oxford University, Auburn University, and DuckDuckGo.
  • Platforms spanning educational portals, AI/SaaS companies, media outlets, fintech firms, security websites, and personal blogs.

The compromise of high-profile domains illustrates both the scale of the attack and the consequences of missing a critical CMS update.

Attack Chain

Every phase of the attack relies strictly on the exposed Ghost CMS environment:

  1. SQL Injection Exploitation: Attackers exploit CVE-2026-26980 to perform an unauthenticated SQL query and steal the admin API keys from the database.
  2. Privileged Access: Using the stolen keys, the attackers leverage Ghost’s admin API to gain management control over users, articles, and themes.
  3. JavaScript Injection: The actors modify article content to embed a lightweight JavaScript loader on legitimate pages.
  4. Second-Stage Cloaking Script: The injected loader fetches additional code from attacker infrastructure. This second-stage code is a cloaking script that fingerprints each visitor—checking browser environment and other characteristics—to decide whether to proceed.
  5. ClickFix Prompt Delivery: If the visitor passes the cloaking checks, the script overlays a fake Cloudflare browser verification page via an iframe on top of the article.
  6. Social Engineering Payload: The overlay instructs the user to prove they are human by pasting a command into the Windows Command Prompt. Executing that command drops the final malware payload.

Payloads and Malware Diversity

Multiple distinct payloads have been observed in this campaign:

  • DLL loaders
  • JavaScript droppers
  • An Electron-based malware sample, tracked as UtilifySetup.exe

The variety of final-stage tools points to a flexible infrastructure that can deliver different malware families based on the attacker’s objectives.

Competing Threat Actor Activity

The article highlights that the exploitation landscape is not controlled by a single group:

  • SentinelOne previously documented at least two distinct activity clusters targeting vulnerable Ghost sites.
  • XLab’s observations confirm that multiple actors are active. In some cases, one actor cleans the injected script left by another and then injects its own script.
  • This behavior shows that compromised Ghost sites are being contested by separate adversaries, each trying to maintain their access.

Targeted Entities and Impact

Beyond the sheer number of domains, the list of known victims underscores the seriousness of the incident:

  • Educational institutions: Harvard University, Oxford University, Auburn University.
  • Privacy-oriented search engine: DuckDuckGo.
  • Other sectors: AI/SaaS, fintech, media, security companies.

No further details about data accessed or additional harm are disclosed in the article; the primary impact described is the injection of malicious scripts leading to malware delivery.

Mitigation and Recommended Response

The article provides several explicit mitigation steps grounded in the researchers’ findings:

  • Immediate upgrade: Upgrade Ghost CMS to version 6.19.1 or any later release.
  • Key rotation: Rotate all admin API keys because any previously used key may have been exposed.
  • Site cleanup: Review the entire website for injected malicious scripts, using the indicators of compromise (IoCs) supplied by XLab.
  • Log retention: Maintain a 30-day record of admin API call logs to enable retrospective investigation and identify when a compromise may have occurred.

Indicators of Compromise (IoCs)

The article states that XLab published a list of IoCs, including injected scripts. While the specific IoCs are not enumerated in the article text, administrators are advised to look for unfamiliar JavaScript loaders injected into articles and to check for connections to the attacker’s second-stage infrastructure.

Context: Broader Ecossytem of Exploited Vulnerabilities

The article’s related articles section references other active exploitation campaigns, including:

  • A critical SQL injection flaw in Drupal being targeted in attacks.
  • A pre-authentication SQL injection in LiteLLM actively exploited.
  • Hackers bypassing SonicWall VPN MFA due to incomplete patching.
  • An auth bypass flaw in the Burst Statistics WordPress plugin under exploitation.
  • A critical Weaver E-cology bug exploited since March.

While these are separate incidents, they collectively underscore a pattern of threat actors rapidly capitalizing on published vulnerabilities before organizations patch.

  • Multiple competing threat actors are re-infecting the same vulnerable Ghost sites, with one group sometimes cleaning another's malicious script to inject its own.
  • High-profile and security-related sites like DuckDuckGo and university portals were compromised, demonstrating that even organizations typically expected to have strong security postures failed to apply the update for over two months.
  • The attackers deployed a diverse set of payloads including DLL loaders, JavaScript droppers, and an Electron-based malware sample, indicating a flexible, multi-stage delivery infrastructure rather than reliance on a single tool.
  • Maintaining a 30-day record of admin API call logs is specifically recommended to enable reliable retrospective investigation of exploitation, beyond just patching and key rotation.

01 Ghost CMS SQL Injection and ClickFix Malware Delivery

A critical SQL injection vulnerability, tracked as CVE-2026-26980, affecting Ghost CMS versions 3.24.0 through 6.19.0 is being actively exploited in the wild. The flaw allows unauthenticated remote attackers to read arbitrary data from the Ghost database, including API keys used for administrative access. Within hours of public disclosure, defenders observed a large-scale campaign leveraging this access to compromise more than 700 domains, among them sites belonging to Harvard University, Oxford University, Auburn University, and the privacy-focused search engine DuckDuckGo.

Once attackers obtain valid admin API keys, they inject malicious JavaScript payloads directly into Ghost-hosted articles. The injected script acts as a cloaking mechanism, evaluating the visitor’s environment before displaying a fake Cloudflare verification prompt. This social engineering technique, known as ClickFix, instructs the victim to execute a command – typically via the Windows Run dialog or a terminal – under the belief they are completing a security check. The executed command retrieves and runs malware from the attacker’s infrastructure, completing the infection chain.

The campaign demonstrates how trusted sites can become malware distribution vectors without any direct breach of end-user accounts. By seeding malicious content into legitimate websites that rank well and enjoy strong reputational trust, the operators bypass typical domain and URL reputation checks. The use of Ghost’s API key theft as the initial access vector also highlights the risk posed by unpatched self-hosted or managed CMS instances, especially in academic and technology sectors that frequently operate public-facing blogs and documentation sites. Ghost version 6.19.1 has been released to correct the vulnerability, but users must additionally rotate all previously generated API keys and audit published content to fully remove injected scripts and prevent re-compromise.
