Executive Summary

Ivanti has issued a critical security advisory regarding Endpoint Manager Mobile (EPMM), urging immediate patching to mitigate CVE-2026-6973. This high-severity remote code execution vulnerability is actively exploited in zero-day attacks and stems from improper input validation. Organizations must update to specific patched versions and rotate administrative credentials to prevent exploitation.

Vulnerability Details and Mitigation

The primary flaw, CVE-2026-6973, requires administrative privileges to execute arbitrary code on vulnerable EPMM systems running version 12.8.0.0 or earlier. Ivanti has released updated versions 12.6.1.1, 12.7.0.1, and 12.8.0.1 as the official mitigation paths. The company explicitly states that the vulnerability only affects the on-premises EPMM product and does not impact cloud-based solutions like Ivanti Neurons for MDM, Ivanti EPM, or Ivanti Sentry. At the time of disclosure, exploitation was reported as very limited. Security teams are advised to review and rotate credentials for all accounts with administrative rights to significantly reduce the attack surface.

Additional Patched Vulnerabilities

Alongside the zero-day, Ivanti addressed four other high-severity EPMM flaws:

  • CVE-2026-5786
  • CVE-2026-5787
  • CVE-2026-5788
  • CVE-2026-7821

These vulnerabilities could enable attackers to gain administrative access, impersonate registered Sentry hosts to obtain valid certificates, invoke arbitrary methods, or access restricted information. There is currently no evidence of wild exploitation for these flaws. However, CVE-2026-7821 is notable because it can be exploited without privileges, specifically affecting users who have configured Apple Device Enrollment.

Historical Context and Regulatory Response

This advisory follows a January disclosure of two critical EPMM code-injection vulnerabilities, CVE-2026-1281 and CVE-2026-1340, which were also exploited in zero-day attacks. Ivanti noted that organizations that rotated credentials in response to the January advisory face significantly reduced risk from the current CVE-2026-6973. The U.S. Cybersecurity and Infrastructure Security Agency (CISA) previously issued an emergency directive, ordering federal agencies to patch CVE-2026-1340 within four days.

The Modern DLP Blind Spot

Traditional data loss prevention strategies have historically treated security as an endpoint or network problem. Organizations deploy agents, inspect files, and monitor traffic, assuming this provides comprehensive coverage. However, recent analysis indicates that 46% of sensitive file uploads to web applications are sent to unsanctioned accounts. This statistic exposes a critical gap in how organizations monitor and control data movement across their digital ecosystems.

Why Traditional DLP Fails

Enterprise workflows have fundamentally shifted from desktop software to browser-based applications. Employees routinely use platforms like Google Workspace, Microsoft 365, Salesforce, GitHub, Jira, and AI tools such as ChatGPT. Instead of downloading and re-uploading files, users interact with data directly within the browser.

This includes copying data between applications, uploading files to various tools, and inputting information into web forms and AI prompts. Compounding this risk is the widespread use of personal accounts and unsanctioned instances without restriction. Traditional DLP controls are not instrumented to monitor these browser-based activities.

How Sensitive Data Leaves the Browser

Data leakage in modern environments occurs through several specific browser activities:

  • Copy and Paste: Users copy sensitive records, credentials, and source code from internal systems and paste them into personal email, SaaS apps, and AI tools. The clipboard acts as a high-risk channel that traditional DLP cannot inspect with necessary context.
  • Form Inputs and AI Prompts: Sensitive data is frequently typed directly into web forms, SaaS applications, or AI prompts. Operating solely within the browser session, endpoint and network DLP controls never trigger.
  • File Uploads to SaaS and AI Tools: File uploads remain a major data loss vector. Employees upload source code, financial data, and customer records to various tools. Up to half of these uploads may target unsanctioned destinations, including personal accounts or unapproved tools.
  • Shadow Accounts and Instances: Risk persists even within approved domains. Users may upload sensitive records to AI prompts using personal accounts or store files in personal cloud storage instead of corporate repositories. Traditional DLP cannot distinguish this activity from normal usage.

Real-World Example and Traditional DLP Limitations

A developer accessing a private repository can copy proprietary code and paste it into a personal ChatGPT session to troubleshoot an issue. No file is downloaded or uploaded, and traffic to the AI tool is permitted, so network protection remains silent. Traditional DLP does not flag the paste action, making the sequence appear as benign activity despite introducing real risk.

Traditional DLP solutions were designed for a different risk model focusing on files at rest and data in motion across endpoints and networks. Endpoint DLP lacks visibility into browser copy/paste activities, web application interactions, and user account types. Network DLP lacks the same context, even with proxy inspection, and distributed workforces exacerbate visibility issues. Cloud DLP provides governance over specific sanctioned instances but cannot inspect unsanctioned browser sessions.

Browser-Native DLP: Closing the Gap

Browser-native DLP operates directly within users’ browsing sessions, providing unique visibility and control. These solutions enable organizations to inspect data in real time across copy/paste activities, form inputs, and file uploads. They understand critical context, including the application in use, the account type, and the data classification.

Security teams can enforce inline controls that block or warn on risky actions based on conditional policies. This approach complements existing DLP stacks by filling a glaring visibility gap that network and endpoint tools cannot address. Solutions like Keep Aware analyze data at the point of user interaction, applying inline enforcement policies to block sensitive actions or reinforce acceptable use policies. This turns previously invisible browser behavior into clear, actionable security signals with robust forensic evidence collection.
