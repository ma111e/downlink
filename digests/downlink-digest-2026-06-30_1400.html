In plain words

Criminals are breaking into internet-exposed Langflow AI app servers by using a serious flaw that lets them run their own code without logging in. They then install software that secretly mines Monero cryptocurrency, shuts off some protections, hides traces, and can spread to other machines that the compromised server can access through saved SSH keys. This matters because AI application infrastructure can become a new doorway into company networks, even when the immediate goal is just stealing computing power for cryptocurrency mining.

Summary
Attackers are actively exploiting a critical unauthenticated Langflow RCE flaw to compromise exposed AI application endpoints and deploy a Monero miner. The malware also disables security controls, removes rival miners, establishes persistence, and can spread to other systems via reused SSH keys.

Threat actors are exploiting CVE-2026-33017, a critical unauthenticated remote code execution vulnerability in Langflow, to deliver a Monero cryptocurrency miner. Trend Micro observed the campaign over a 19-day window between March 27 and April 15, 2026. The activity targets exposed artificial intelligence application endpoints and uses them as an initial access path into enterprise networks.

The attack runs attacker-supplied Python code inside an unauthenticated Langflow API endpoint to download and execute a shell script. That script checks for a binary called “lambsys,” downloads it with curl or wget if needed, launches it as a detached process, and spreads to SSH-reachable hosts the victim can authenticate to. The Go-based binary disables security controls including AppArmor, Ubuntu’s Uncomplicated Firewall, iptables, SELinux, the kernel NMI watchdog, and Alibaba Cloud’s Aliyun agent.

The malware terminates competing cryptocurrency miners linked to Kinsing, WatchDog, Rocke, and Outlaw, removes rival wallet and key material, deletes system logs, and establishes cron-based persistence. It contacts an external server at 83.142.209[.]214:80 to fetch a TAR archive containing a bespoke XMRig miner, then queries ipinfo[.]io for the host’s public IP address and location. Trend Micro said the campaign shows how exposed AI application endpoints are becoming another route into enterprise environments.

  • The campaign shows that exposed AI application endpoints are being used as an initial access path into enterprise environments, not just as isolated targets for cryptomining.
  • Lambsys is built to remove competing miners and rival wallet or key material, indicating the operators expect infected systems may already be contested by other cryptojacking groups.
  • The malware’s use of many short-lived shell subprocesses favors reliability over stealth, allowing individual command failures to occur without stopping the rest of the attack logic.
  • The request to ipinfo.io supports operational decisions such as selecting nearby mining pools to improve hash rate and excluding victims in certain regions.
  • Trend Micro found a related binary artifact compiled in May 2024, suggesting the operators have been iterating on this malware family for more than two years.

Summary

The campaign is notable for using blockchain-based C2 lookup and stealthy Chromium extension installation, making takedown and detection harder.

Cybersecurity researchers have flagged an active browser extension campaign called Silent Swap that is designed to steal cryptocurrency by replacing wallet addresses when users initiate transactions. McAfee Labs said the campaign is delivered through unsigned installers observed in .NET and Golang variants, which deploy a malicious Chromium extension pretending to be a benign “Google Notes” utility. The extension requests access to the clipboard, all URLs, and browsing history so it can intercept and manipulate copied wallet addresses.

The campaign affects Chromium-based browsers such as Google Chrome, Microsoft Edge, Brave, and Vivaldi by modifying browser Secure Preferences and Preferences files to load the extension without normal store-based installation. McAfee said the malware recalculates and updates browser security verification values after tampering with these files, making the browser treat the extension as legitimately installed. The activity also uses EtherHiding, which relies on a blockchain smart contract to retrieve active command-and-control server details that operators can rotate with a single transaction.

Silent Swap maps wallet addresses for Bitcoin, Ethereum, Bitcoin Cash, Ripple, and Dash to attacker-controlled addresses on the server side, while submitted Solana addresses resolve to a single attacker address. McAfee reported infections across multiple countries, with a higher concentration in India and other impacted countries including the U.S., Brazil, Indonesia, and Spain. The article also describes Socket’s findings on malicious Chrome and Firefox extensions named “VPN Go: Free VPN,” which provide visible proxy functionality while monitoring and exfiltrating clipboard contents such as passwords, authentication codes, API keys, OAuth tokens, and seed phrases.

  • Silent Swap’s use of a blockchain smart contract as a lookup point makes the command-and-control infrastructure easier for operators to rotate, because they can change the active server with a single transaction instead of modifying the malware.
  • The malware recalculates browser security verification values after modifying Chromium settings files, which helps the malicious extension appear legitimately installed and bypass the normal extension store approval flow.
  • The wallet replacement logic is server-side and deterministic, returning the same attacker-controlled replacement address when the same victim address is submitted again.
  • The VPN Go extensions used a staged update pattern, first appearing benign in official extension stores before later versions introduced clipboard-stealing behavior.
  • Users who installed the VPN Go extensions are advised to treat any secrets copied while the extensions were active as compromised, because the clipboard theft was not limited to cryptocurrency wallet addresses.

In plain words

Researchers at Adversa AI found that many open-source AI coding helpers can be fooled into running harmful computer commands by hiding them in ways the safety check does not recognize, even though the computer still runs them normally. People who use these tools are affected because the tools can run commands with the same access as the user, meaning a poisoned project or package could delete files or steal things like SSH keys and cloud credentials. The broader concern is that these assistants may act on untrusted text before they truly understand what the computer will do, especially when they are allowed to run tasks automatically.

Summary
Adversa AI’s GuardFall research shows that most tested open-source AI coding agents can be tricked into running dangerous shell commands because their safety filters check raw text instead of what bash actually executes. Users should avoid auto-execution and isolate agents from real home directories and secrets until stronger command parsing protections are added.

Adversa AI research named GuardFall found that a decades-old shell trick can bypass safety checks in many open-source AI coding and computer-use agents. The bypass worked against ten of the eleven popular agents the firm tested, with only Continue built to defend against it. The affected tools named in the article include opencode, Goose, Cline, Roo-Code, Aider, Plandex, Open Interpreter, OpenHands, SWE-agent, and Hermes.

The issue matters because these agents can run shell commands with the user’s full account access. If an agent is pointed at a booby-trapped repository or software package, hidden instructions could lead it to run commands that wipe files or steal secrets such as SSH keys, cloud credentials, or other data in the user’s home folder. Adversa describes the work as lab research, and the article says no public exploitation has been reported.

The technical problem is that many agents check commands as plain text against blocklists, while bash rewrites the command before running it by stripping quotes and expanding shortcuts. For example, a filter watching for rm may not detect r''m, but bash removes the empty quotes and runs rm anyway. The article recommends reducing exposure by using a throwaway home folder, turning off auto-execute modes unless necessary, avoiding agents on pull requests from forks, and treating repository configuration files as untrusted code.

  • Adding more blocked command patterns is not a complete fix because the issue is described as a broader unsafe design pattern where the guard and bash interpret the same command differently.
  • A malicious command does not need to be presented directly to the AI; the article says it can be hidden inside normal-looking material such as a build file or tool documentation and then emitted as a routine step.
  • The strongest tested defense was to parse the command the way bash will before deciding whether to run it, and Adversa said this design is portable and could be reimplemented by an experienced engineer in about two days.
  • Even the agent that performed best, Continue, had weaker protection in its command-line auto-run mode, showing that safety can vary significantly between modes within the same tool.
  • Repository-supplied configuration files such as .aider.conf.yml are especially risky because the article says a malicious one can trigger the attack on the first accepted edit.
Key Points

  • 02The bypass works because many agents check commands as plain text against blocklists, while bash later rewrites the text by stripping quotes and expanding shortcuts before execution.
  • 03An affected agent could run destructive or secret-stealing commands with the user’s full account access if it is pointed at a booby-trapped repository or software package.
  • 04Adversa reported that opencode, Goose, Cline, Roo-Code, Aider, Plandex, Open Interpreter, OpenHands, SWE-agent, and Hermes left the gap open, while Continue’s default editor mode blocked the tested payloads.
