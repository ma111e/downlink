Device-code phishing method

Device-code phishing is described as an attack in which threat actors send a device authorization request to the target service’s provider and forward the generated code to the victim. The victim is tricked into entering the code on the service’s legitimate login page. In this case, the target service is Microsoft 365, and the legitimate login destination is microsoft.com/devicelogin.

The article states that entering the code authorizes the attacker to register a rogue device with the victim’s Microsoft 365 account. This gives the attacker unrestricted access to the victim’s data and services. The article specifically names email, calendar, and cloud file storage as affected Microsoft 365 services.

The article also notes that device-code phishing has increased sharply this year. Push Security recently warned that this type of attack increased by 37x and is supported by at least ten distinct phishing-as-a-service platforms and private kits. A Proofpoint report is also cited as recording a similar surge in the use of the tactic.

Tycoon2FA campaign flow

According to eSentire, the attack begins when the victim clicks a Trustifi click-tracking URL in a lure email. The campaign observed by eSentire used an invoice-themed phishing email. The redirect chain moves through Trustifi, Cloudflare Workers, and several obfuscated JavaScript layers.

The victim eventually lands on a fake Microsoft CAPTCHA page. From there, the phishing page retrieves a Microsoft OAuth device code from the attacker’s backend. The victim is instructed to copy and paste that code into microsoft.com/devicelogin.

After entering the code, the victim completes multi-factor authentication on their end. The article states that Microsoft then issues OAuth access and refresh tokens to the attacker-controlled device. This means the attack relies on the victim interacting with Microsoft’s legitimate device-login flow rather than only a fake credential page.
