In plain words

This article is about an ongoing scam that uses hacked WhatsApp accounts to send fake business or money-related files to people in many countries. If someone opens one of these files on a Windows computer, it can quietly install software that gives the attacker control of that computer from far away. It matters because the messages can come from real contacts, making them look trustworthy, and the problem has been seen across countries including Brazil, India, Mexico, Singapore, the UK, Spain, Taiwan, Australia, Russia, Vietnam, and Malaysia.

Summary
A global phishing campaign is using compromised WhatsApp accounts to send fake business-document VBScript files that, when opened on Windows, silently install ManageEngine Endpoint Central for attacker-controlled remote access. Users should be extremely cautious with unexpected files sent over WhatsApp, even from trusted contacts, and verify them before opening.

An ongoing malware campaign is targeting WhatsApp users in multiple countries with deceptive messages that deliver heavily obfuscated VBScript files. The files are disguised as business and financial documents and are sent from compromised WhatsApp accounts to the contacts of those users. Kaspersky telemetry shows activity in Brazil, India, Mexico, Singapore, the UK, Spain, Taiwan, Australia, Russia, Vietnam, and Malaysia. The article says the exact method used to compromise the WhatsApp accounts is still unknown.

If a recipient downloads and opens the file on Windows, it starts an infection chain that fetches additional scripts from attacker-controlled infrastructure. Those scripts disable UAC protections through Registry modifications and then download a ZIP archive containing ManageEngine Endpoint Central. The software is silently installed and configured to connect to attacker-controlled management servers, giving the attackers remote administration access to the victim’s computer. Kaspersky also notes that files delivered via WhatsApp Web must be downloaded, while in the WhatsApp Desktop client they can be executed directly by Windows Script Host.

Kaspersky did not attribute the campaign to a specific threat actor, although researchers observed signs of Chinese language use and infrastructure overlap with IPs previously associated with ValleyRAT and Gh0st RAT activity. The article says there is not enough evidence for high-confidence attribution. It advises WhatsApp users to be cautious even with files sent by trusted contacts and to verify suspicious files through secondary means. It also recommends scanning downloaded files with up-to-date antivirus software before executing them.

  • The campaign relies on messages sent from already compromised WhatsApp accounts, which makes the malicious files appear to come from trusted contacts rather than unknown senders.
  • Attackers are using a legitimate IT administration product, ManageEngine Endpoint Central, as the final payload, showing that normal remote-management software can be repurposed to provide unauthorized access.
  • The delivery path changes the risk: files sent through WhatsApp Web have to be downloaded before use, while in the WhatsApp Desktop client they can be executed directly by Windows Script Host.
  • The use of localized filenames for fake financial and business documents indicates the campaign was tailored for targets in multiple countries rather than a single region.
