Sygnia researchers reported that the Chinese-linked Velvet Ant threat group carried out a decade-long espionage operation against the isolated critical infrastructure network of a large organization. The campaign, named Operation Highland, began in 2016 and targeted vulnerable internet-facing systems before pivoting into an “air-gapped” environment with no direct external path. The article describes the operation as giving the attackers full visibility into administrative activity.

The attack started with compromised internet-facing servers, although the specific product or vulnerability used was not disclosed. Velvet Ant deployed a modified GS-Netcat reverse shell disguised as a legitimate system component, which connected to a hardcoded relay domain and provided encrypted remote shell access. The reverse shell persisted through either a malicious systemd service or startup script modification.

The attackers then installed a custom SOCKS5 proxy to tunnel network traffic and reach internal systems that were not directly accessible from the internet. The proxy ran as a daemon masquerading as “smbd -D,” with different filenames and ports on each host. These compromised systems became internal pivot points for further access.

A key part of the intrusion was the construction of a remote-execution path into the isolated network. Velvet Ant modified the configuration of a compromised internet-facing Nginx server to proxy specially crafted requests to a compromised backend server, then altered the backend server’s Nginx configuration to forward requests to a FastCGI process. That FastCGI wrapper launched a custom binary named “uptime,” which established SSH connections to isolated-network systems using parameters supplied in HTTP POST requests.

After establishing access, Velvet Ant targeted authentication mechanisms for persistence and credential theft. The group replaced legitimate pam_unix.so modules with backdoored versions that accepted hardcoded passwords and harvested credentials, and Sygnia identified nine distinct variants of the malicious PAM module. The attackers also replaced OpenSSH components such as ssh, sshd, and scp with trojanized versions that captured credentials, logged commands, and stored collected data locally for retrieval.

Sygnia said cleanup was particularly difficult because the attackers had replaced many critical components, and removing them risked breaking authentication, locking out administrators, and causing operational outages. The researchers built a testing lab, validated the binary replacement process, profiled hosts, tested results, and prepared rollback procedures before remediation. Sygnia recommended treating authentication components such as PAM, OpenSSH, and Windows LSASS as critical security assets and protecting them with EDR, file integrity monitoring, hardened privileged access, MFA, continuous monitoring, and tested offline recovery plans.

Overview

Sygnia researchers reported a long-running cyber-espionage campaign by the Chinese-linked Velvet Ant activity cluster against the isolated critical infrastructure network of a large organization. The campaign, which Sygnia dubbed Operation Highland, began in 2016 and continued for 10 years. The attackers first compromised vulnerable internet-facing systems and then pivoted into an environment described as “air-gapped,” with no direct internet connection.

The article highlights that Velvet Ant did not merely maintain access through a single compromised host. Instead, the group took control of parts of the target organization’s authentication stack, giving it visibility into administrative activity and a way to preserve access even as passwords changed or sessions ended.

Actor and Context

The intrusion is attributed to Velvet Ant, a Chinese cyberespionage threat group or activity cluster. Sygnia previously documented lengthy Velvet Ant espionage operations in 2024, including activity targeting F5 BIG-IP devices that reportedly operated undetected for three years. Also in 2024, Cisco warned of a zero-day in NX-OS running on Nexus switches that Velvet Ant exploited to gain access to targets.

In Operation Highland, the victim is described as a large organization with an isolated critical infrastructure network. The article does not name the organization and does not identify the specific internet-facing product or vulnerability used in the initial compromise.

Initial Access and External Foothold

The attack began with the compromise of internet-facing servers. Sygnia did not specify the product or vulnerability involved in that first stage.

Velvet Ant deployed a modified GS-Netcat reverse shell disguised as a legitimate system component. This component connected to a hardcoded relay domain and provided encrypted remote shell access. The reverse shell achieved persistence through either a malicious systemd service or startup script modification.

Internal Pivoting Through SOCKS5 Proxying

After establishing the initial foothold, Velvet Ant installed a custom SOCKS5 proxy to tunnel network traffic. This allowed the attackers to reach internal systems that were not directly accessible from the internet.

The proxy ran as a daemon masquerading as “smbd -D.” It used different filenames and ports on each host, turning compromised servers into internal pivot points.

Remote Execution Path Into the Isolated Network

A central technical element of Operation Highland was the creation of a remote-execution path into the segregated environment. Velvet Ant modified a compromised internet-facing Nginx server so that specially crafted requests would be proxied to a compromised backend server.

The backend server’s Nginx configuration was also altered. It forwarded requests to a FastCGI process, fcgiwrap, listening on a separate port. The FastCGI wrapper acted as an execution bridge by processing requests and launching a custom binary named “uptime.”

The “uptime” tool established SSH connections to systems within the isolated critical infrastructure network. It used parameters supplied in HTTP POST requests. Sygnia said this chain allowed Velvet Ant to establish a remote-execution path into the segregated environment through simple HTTP requests, without requiring any direct connection to the critical infrastructure network.

Authentication Stack Compromise

Once Velvet Ant had access to the isolated environment, the attackers focused on long-term persistence and credential theft by targeting Linux Pluggable Authentication Modules, or PAM. PAM is described in the article as a set of libraries that lets administrators configure authentication methods.

The attackers replaced legitimate “pam_unix.so” modules with backdoored versions. These malicious modules accepted hardcoded passwords and harvested user credentials. Sygnia identified nine distinct variants of the malicious PAM module, each compiled in a separate build environment.

Sygnia said two of the malicious PAM modules stood out: one acted only as a backdoor, and another collected credentials.

OpenSSH Component Replacement

Velvet Ant also replaced OpenSSH components with trojanized versions. The altered components included ssh, sshd, and scp.

These trojanized OpenSSH binaries captured credentials, logged commands entered during SSH sessions, and stored the collected data locally for later retrieval. Sygnia said this gave the attackers access to credentials as they were used in the target environment and allowed them to bypass the authentication flow.

According to the article, this meant administrative activity became fully observable, including every login and every command executed across compromised hosts. Access was no longer tied to a single foothold but embedded into the authentication process itself.

Impact on Persistence and Containment

The authentication-level compromise helped Velvet Ant maintain persistence despite password changes and session terminations. Sygnia said the approach reduced the effectiveness of conventional containment measures.

Because the attackers had modified PAM and OpenSSH components, access was tied to the systems that mediated authentication and administrative activity. This made the compromise especially difficult to remediate safely.

Cleanup and Recovery Challenges

Sygnia said removing Velvet Ant from the environment was particularly complicated after the compromise was discovered. The attackers had replaced so many critical components with custom versions that removing them risked breaking authentication, locking legitimate administrators out, and causing operational outages.

To address this, Sygnia built a testing lab to validate the binary replacement process. The researchers profiled each host, tested results, and prepared rollback procedures before attempting cleanup.

Defensive Recommendations From Sygnia

Sygnia recommended treating authentication components as critical security assets. The article specifically names PAM, OpenSSH, and Windows LSASS as components that should be protected.

Recommended protections included:

  • Endpoint detection and response coverage.
  • File integrity monitoring.
  • Hardened privileged access.
  • Multi-factor authentication.
  • Continuous monitoring for unauthorized modifications.

Sygnia also recommended that organizations plan for offline recovery. This includes strict backups, an appropriate schedule for automatically creating snapshots, and immutable copies. The restoration process should include tested backups, recovery hosts running validated operating systems, and validated recovery scripts.

  • Treating PAM, OpenSSH, and Windows LSASS as critical security assets is essential because compromising authentication components can make password changes and session termination ineffective containment measures.
  • Air-gapped or segregated environments can still be reachable when compromised internet-facing and backend systems are chained into a proxy-based execution path.
  • File integrity monitoring and EDR coverage should include authentication binaries and libraries, not only conventional malware locations, because Velvet Ant embedded persistence directly into trusted system components.
  • Remediation of authentication-stack compromises should be rehearsed in a lab with host profiling, validation, and rollback procedures because removing trojanized components can break access for legitimate administrators.
  • Immutable backups and tested offline recovery processes are important because restoring trusted operating system components may be safer than attempting ad hoc cleanup on heavily modified hosts.

EXEC Executive Overview

This digest centers on a long-running Chinese-linked espionage operation targeting an isolated critical infrastructure network. Reporting from Sygnia describes how Velvet Ant sustained access for approximately 10 years by compromising authentication paths, modifying trusted Linux components, and creating covert remote execution routes into a segregated environment.

The case highlights a mature persistence model focused less on noisy malware deployment and more on deep integration with identity, access, and network control planes. For technical defenders and executives alike, the incident illustrates how isolation alone can be undermined when authentication flows and administrative infrastructure are subverted.

01 Chinese-Linked Espionage in Critical Infrastructure

Sygnia attributed the campaign, dubbed Operation Highland, to the Chinese-linked Velvet Ant threat group. The activity reportedly affected a large organization’s isolated critical infrastructure network, where the attackers maintained espionage access for a decade. The operation began with compromise of internet-facing systems before the attackers pivoted inward using a modified GS-Netcat reverse shell and a custom SOCKS5 proxy.

The campaign’s most significant feature was the abuse of authentication and application infrastructure to reach systems that were not directly exposed. Velvet Ant modified Nginx and FastCGI configurations to create a remote-execution pathway into the segregated environment through crafted HTTP requests. This allowed interaction with the isolated network without requiring a direct connection to the critical infrastructure segment.

Sygnia also reported that the attackers replaced legitimate Linux PAM modules and OpenSSH components with trojanized versions. These backdoored components accepted hardcoded passwords, harvested credentials, captured commands, and preserved persistent access. By embedding themselves into core authentication mechanisms, the operators gained durable control over administrative pathways while blending into expected system behavior.

02 Persistence Through Trusted System Components

The operation demonstrates a persistence strategy built around the compromise of trusted binaries and configuration layers rather than reliance on standalone implants alone. Backdoored PAM and OpenSSH components gave Velvet Ant access at the point where users and administrators authenticated, creating opportunities to collect credentials and observe operator activity. The use of modified web and FastCGI configurations further expanded the attackers’ ability to execute commands through infrastructure that would normally be considered part of legitimate service delivery.

Remediation was described as complex because the compromised components were central to system authentication and remote administration. Sygnia noted that replacing many trojanized critical components could risk breaking authentication, locking out legitimate administrators, or causing operational outages. In the context of critical infrastructure, that operational fragility made eradication more difficult than simply removing a conventional malware payload.

Overall, Operation Highland reflects an espionage pattern in which adversaries target the connective tissue between exposed enterprise systems and protected operational environments. The reported decade-long access underscores the strategic value of authentication infrastructure, reverse proxies, and administrative services as targets for stealthy persistence inside high-value networks.
