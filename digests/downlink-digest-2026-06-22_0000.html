In plain words

They rented powerful supercomputers to unscramble the collected passwords and sold that access to other criminals. Any organization with a compromised device could now have intruders inside its network, putting sensitive data and entire operations at risk.

Summary
The FortiBleed campaign uses a custom sniffer tool (FortigateSniffer) deployed on compromised FortiGate firewalls to capture authentication credentials from network traffic, which attackers then crack with a rented GPU cluster to gain network access.

The SOCRadar report details an active and large-scale campaign called FortiBleed that targets Fortinet FortiGate firewalls to harvest authentication credentials. The threat actor, functioning as an initial access broker, has been compromising devices since at least February 2026 through credential stuffing and brute-force attacks, gaining administrative access to over 430,000 firewalls worldwide. Once inside, the attackers deploy a custom Go-based sniffer named FortigateSniffer, which exploits the FortiOS ‘diagnose sniffer packet’ feature to eavesdrop on network traffic.

The sniffer is configured to monitor 24 protocols including Kerberos, LDAP, SMB, RADIUS, RDP, and various email and database services, capturing cleartext credentials and password hashes directly from live traffic. Captured packets are converted to PCAP files by a component called SNIFTRAN and then fed into a Python-based toolkit that extracts hashes, Kerberos tickets, and plaintext login details. The extracted NTLM and Kerberos hashes are cracked using Hashcat on a rented cluster of 36 enterprise-class GPUs, repurposing cloud AI infrastructure for large-scale credential theft.

The operation has already exposed VPN credentials for tens of thousands of FortiGate devices, and researcher Kevin Beaumont has published a list of attacker IP addresses to help organizations identify potential compromise. The campaign highlights how a built-in diagnostic tool can be weaponized to turn a perimeter firewall into a stealthy network wiretap, empowering downstream attacks and account takeovers across compromised corporate environments.

  • Attackers rented a cluster of 36 enterprise GPUs from a GenAI cloud provider to crack stolen credentials, repurposing AI infrastructure for large-scale password cracking rather than legitimate workloads.
  • By abusing the legitimate FortiOS 'diagnose sniffer packet' feature, the threat actors turned compromised firewalls into stealthy network wiretaps capable of capturing authentication secrets from 24 different protocols.
  • The sniffer's parsing toolkit extracts not only NTLM and Kerberos hashes but also cleartext email credentials (SMTP/IMAP/POP3) and database credentials (MySQL, PostgreSQL, MSSQL), exposing any cleartext authentication flowing through the firewall.
  • The release of attacker IP addresses by researcher Kevin Beaumont gives defenders a concrete, immediate method to search historical firewall logs for evidence of past compromise even if they never observed the sniffer running.

In plain words

A serious programming flaw was found in a video processing tool called FFmpeg, which is built into countless apps that play, organize, or preview videos. A crafty video file can abuse this flaw, and when any affected app automatically looks at it—even just to generate a tiny preview—it can crash the program or let an attacker secretly run commands on your computer. Because FFmpeg is so widespread, this puts media servers, file managers, cloud storage, and even AI tools at risk unless they are updated or the vulnerable feature is turned off.

Summary
A critical heap out-of-bounds write vulnerability (CVE-2026-8461, PixelSmash) in FFmpeg's MagicYUV decoder enables remote code execution via a crafted video file, impacting virtually any media-processing application that links FFmpeg, including Jellyfin, Nextcloud, and desktop players.

JFrog Security Research discovered and disclosed CVE-2026-8461, named PixelSmash, a critical heap out-of-bounds write vulnerability in FFmpeg’s MagicYUV decoder. Triggered by a crafted video file with a malformed slice height, the bug can cause immediate crashes in any application that links FFmpeg’s libavcodec—from desktop players to server-side media pipelines. The researchers escalated the flaw from a denial-of-service to reliable remote code execution, demonstrating full compromise of both a Jellyfin media server and a Nextcloud instance simply by uploading a 50 KB AVI file.

The root cause is a rounding mismatch: when the attacker-controlled slice height is odd, ceiling-rounded shifts allocate one extra chroma row per slice, eventually writing past the end of the heap buffer. The exploit carefully overwrites an AVBuffer management structure’s function pointer to redirect execution to system(), while preserving glibc malloc metadata so the corrupted heap does not abort prematurely. The demonstration required ASLR to be disabled; however, the article notes a separate, long-standing uninitialized heap disclosure in FFmpeg’s FlashSV decoder that could potentially be chained to defeat address randomization in the future.

PixelSmash exemplifies a software supply-chain vulnerability: because the MagicYUV decoder is enabled by default in virtually every FFmpeg build, countless downstream projects—including mpv, Kodi, Jellyfin, Emby, Immich, PhotoPrism, OBS Studio, and even AI/ML frameworks like vLLM—inherit the bug without any explicit opt-in. Real-world attack vectors include zero-click torrent downloads into monitored media libraries and automatic thumbnail generation on file managers or cloud platforms. The only tested application that was immune by design was Plex, which builds FFmpeg with a minimal decoder allow-list. Users are urged to upgrade FFmpeg to the patched version 8.1.2 or disable the MagicYUV decoder at build time.

  • The exploit transforms a one-row heap overflow into arbitrary command execution by overwriting the AVBuffer structure's function pointer to system(), while carefully preserving glibc malloc metadata to avoid early crashes.
  • Jellyfin and Emby process the malicious file with exit code 0 and no error output, meaning the heap corruption is silent and repeated exploitation may go completely undetected by administrators.
