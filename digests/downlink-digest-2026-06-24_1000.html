The actor appears to gain access opportunistically through public-facing application exploits and malicious droppers, then uses persistence, API-hooking evasion, reconnaissance, and credential theft techniques.

Kaspersky uncovered a previously undocumented malware family named SharkLoader while investigating activity affecting a diplomatic organization in Indonesia. The broader campaign, tracked as StrikeShark, uses SharkLoader to deploy Cobalt Strike Beacon on compromised systems. Additional infections were identified across multiple countries and sectors, including government-related entities, software development companies, and organizations in Hong Kong, Lebanon, Syria, Colombia, North Macedonia, Nepal, and Serbia.

The threat actor gained initial access through exploitation of internet-facing applications, including Microsoft Exchange, Microsoft SharePoint, Openfire Server, and GeoServer, as well as through custom droppers disguised as legitimate software such as Google Update and Cisco AnyConnect. Kaspersky assesses with medium confidence that the actor primarily relies on publicly available proof-of-concept exploits, because all vulnerabilities observed during the investigation had public exploit code available. The campaign shows both strategic and opportunistic characteristics, with victims including government and software development organizations as well as a broader set of affected entities.

SharkLoader uses DLL side-loading, encrypted modules, reflective loading, API hooks, and memory protection changes to execute Cobalt Strike Beacon and hinder detection. The infection chain includes a legitimate SystemSettings.exe abused to load a malicious SystemSettings.dll, followed by encrypted components such as DscCoreR.mui and SyncRes.dat. Post-compromise activity included reconnaissance, Active Directory enumeration, use of tools such as FScan, Searchall, Pillager, and SharpGPOAbuse, and attempts to dump credentials from LSASS and the NTDS database.

  • The actor appears to rely primarily on publicly available proof-of-concept exploits rather than custom exploit development, because all vulnerabilities observed in the campaign had public exploit code available.
  • SharkLoader’s dropper-based infections create two scheduled tasks: one that runs every second for immediate execution and is quickly removed, and another that runs every five minutes for longer-term persistence.
  • SharkLoader changes Cobalt Strike-related memory regions from executable permissions to read-write during Sleep calls and restores executable permissions afterward, indicating an explicit attempt to evade memory scanning for executable code regions.
  • The campaign combines opportunistic exploitation of many public-facing applications with victims that include government and software development organizations, creating a mixed pattern that Kaspersky says suggests both strategic and opportunistic characteristics.
  • Although several post-exploitation tools used in the campaign were developed by Chinese-speaking developers, Kaspersky found no code reuse, infrastructure overlap, or operational similarity sufficient to attribute the activity to a known threat actor.

  • The affected workflows could expose especially powerful credentials, including a non-expiring GitHub App key in Microsoft Azure Sentinel and a token with full write permissions in Apache Doris.
  • Novee Security linked the spread of these weaknesses to agentic coding, saying such CI/CD flaws can be reproduced persistently and at scale across repositories.
