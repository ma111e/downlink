Incident Overview TeamPCP hackers are actively threatening to leak Mistral AI’s internal source code unless they secure a buyer within a week. The group is asking for $25,000 for a dataset comprising nearly 450 repositories. They claim the stolen data totals approximately five gigabytes and includes internal code used for model training, fine-tuning, benchmarking, and inference experiments.

The hackers have indicated that the asking price is flexible and they are open to receiving competitive offers. They warn that if no purchase is made within seven days, they will publicly release the entire dataset for free. This commercialized threat strategy prioritizes rapid monetization over permanent data destruction.

Attack Vector and Supply-Chain Impact The breach originated from the Mini Shai-Hulud supply-chain attack, which successfully compromised official npm packages from both TanStack and Mistral AI. Attackers utilized stolen CI/CD credentials and exploited legitimate workflows to infiltrate the systems. This initial compromise rapidly cascaded across hundreds of other software projects hosted on npm and PyPI registries.

Notable third-party projects affected by the spread include UiPath, Guardrails AI, and OpenSearch. TeamPCP successfully contaminated some of Mistral AI’s SDK packages for a brief period before the intrusion was contained. The incident demonstrates how a single compromised developer device can rapidly propagate across multiple software ecosystems.

Company Response and Scope of Compromise Mistral AI confirmed the security incident and published a detailed security advisory outlining the breach timeline. Forensic analysis revealed that the compromised data did not intersect with the company’s core code repositories. The company explicitly stated that hosted services, managed user data, and all research and testing environments remained secure.

The breach was traced back to a single developer device that was impacted by the TanStack supply-chain attack. This distinction highlights that the attackers targeted development artifacts rather than proprietary model weights or production environments. Mistral AI’s rapid forensic response successfully contained the threat to SDK packages without core infrastructure exposure.

Related Industry Impact The TanStack supply-chain attack also compromised OpenAI’s internal systems, granting limited access to source code repositories for two employees. OpenAI confirmed that while a small set of credentials was stolen, there is no evidence they were used to launch additional attacks. OpenAI has already rotated the exposed code-signing certificates and issued a mandatory update warning for macOS users.

Users must update their OpenAI desktop applications before June 12, or the software will fail to launch and stop receiving updates. This urgent patching requirement underscores how credential theft directly impacts end-user deployment reliability and software continuity.

EXEC Executive Overview

The threat actor TeamPCP is currently advertising a collection of approximately 450 Mistral AI code repositories for sale, demanding a payment of $25,000. The group has issued a deadline of one week for a buyer, threatening to publicly leak the data should no transaction occur. This extortion attempt stems from the Mini Shai-Hulud supply-chain attack, a campaign that successfully compromised continuous integration and continuous deployment (CI/CD) credentials associated with Mistral AI. During the intrusion, the attackers briefly contaminated Mistral AI’s SDK packages, indicating a potential for downstream impact on consumers of these libraries. The brief contamination of SDK packages introduces a supply-chain risk where downstream consumers of Mistral AI libraries could inadvertently integrate compromised artifacts into their own build processes. This mechanism highlights the cascading risk inherent in compromised build systems, where a single point of failure in CI/CD infrastructure can affect a wide ecosystem of dependent projects. The attackers’ ability to publish packages suggests deep access to the release pipeline, enabling the distribution of modified code before detection or remediation.

Mistral AI has publicly confirmed the scope of the breach, clarifying that the exfiltrated data did not encompass core code repositories, hosted services, managed user data, or research environments. This distinction suggests the attackers focused on internal development artifacts rather than production systems or sensitive customer information. The Mini Shai-Hulud campaign also intersected with the broader TanStack supply-chain attack. Intelligence indicates that the TanStack compromise impacted OpenAI, resulting in the unauthorized access to a limited subset of internal source code repositories belonging to two employees. The intersection with the TanStack attack reveals a pattern of threat actors leveraging compromised package registries or build tools to pivot between targets. OpenAI’s compromise, limited to internal source code for two employees, indicates a targeted approach within the broader TanStack incident. The restriction of access to specific repositories suggests the attackers may have utilized credential harvesting or misconfigured permissions to isolate sensitive code segments, rather than conducting a broad exfiltration. This granular access pattern aligns with the methodology observed in the Mini Shai-Hulud campaign, where CI/CD credentials provided the primary vector for accessing development environments.

TeamPCP’s actions demonstrate a shift toward monetizing access to internal repositories, leveraging the high value of proprietary code and development workflows. The demand for $25,000 for nearly 450 repositories indicates an assessment of the data’s utility to competitors or researchers. The threat of public release serves as a coercive mechanism to accelerate a sale, leveraging the reputational risk associated with exposing internal code. This behavior reflects an increasing sophistication in threat actor operations, where data theft is coupled with structured extortion campaigns targeting specific organizations. The convergence of Mini Shai-Hulud and TanStack suggests overlapping tactics or shared infrastructure among supply-chain threat actors targeting major AI development organizations. The contamination of SDK packages during the Mini Shai-Hulud attack presents a potential propagation vector, where compromised build artifacts could introduce malicious code into downstream projects. The reported activities reflect a growing trend of threat actors targeting the foundational tools and repositories that underpin modern software development, particularly within the AI sector, where intellectual property and model development code represent high-value targets.
