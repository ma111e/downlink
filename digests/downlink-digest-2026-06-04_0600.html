Overview

CISA added CVE-2026-45247 to its Known Exploited Vulnerabilities catalog after reports of active exploitation in the wild. The vulnerability affects Mirasvit Cache Warmer, a popular Magento full-page cache extension. The flaw is rated critical, with a CVSS score of 9.8.

The issue affects all versions of the extension prior to version 1.11.12. Patches were released on May 25, 2026. In response to active exploitation, Federal Civilian Executive Branch agencies were ordered to apply the fixes by June 6, 2026.

Technical Details

CVE-2026-45247 is described as a deserialization of untrusted data vulnerability. CISA said the flaw could allow unauthenticated attackers to achieve remote code execution by supplying a crafted serialized PHP object in the CacheWarmer cookie.

Sansec said the vulnerability can be exploited through any storefront request that carries a crafted CacheWarmer cookie. The cookie value is deserialized using PHP’s native unserialize() function without requiring authentication or admin privileges. Because the value comes directly from the client, an attacker controls the objects PHP reconstructs.

Sansec characterized the issue as PHP object injection, corresponding to CWE-502. It said that, when combined with a gadget chain from classes already shipped with Magento and its dependencies, object injection can escalate to remote code execution.

Exposure and Affected Parties

Sansec said it identified about 6,000 stores running Mirasvit extensions. The company also said the exact number is likely higher because content delivery networks such as Cloudflare can mask installs.

The affected software is Mirasvit Cache Warmer for Magento. The article identifies gaming and business sites as the primary targets of observed activity. The U.S., the U.K., France, and Australia were named as the most targeted countries.

Observed Exploitation

Imperva disclosed that it observed active attack activity attempting to exploit CVE-2026-45247. The observed attacks used serialized PHP object payloads delivered through malicious HTTP requests.

According to Imperva, observed payloads contained base64-encoded serialized objects designed to trigger PHP object deserialization and achieve remote code execution through commonly abused gadget chains. The payloads attempted to invoke functions such as system() and current() to execute arbitrary commands on the underlying server.

Imperva also said that in several observed cases, attackers used test commands designed to validate successful code execution. The article states that it is currently not known who is behind the exploitation efforts. It also says the apparent end goal is to flag vulnerable Magento environments and confirm that remote code execution is possible.

Detection Guidance Mentioned in the Article

Site owners were advised to audit storefront requests that carry a CacheWarmer cookie whose value contains the marker “CacheWarmer:” followed by a Base64-encoded string.

Sansec added that serialized PHP objects base64-encode to values starting with “Tz,” “Qz,” or “YT.” As a result, a CacheWarmer cookie value matching “CacheWarmer:(Tz|Qz|YT)” is a strong indicator of an exploitation attempt.

Response

CISA’s addition of CVE-2026-45247 to the Known Exploited Vulnerabilities catalog followed reports of active exploitation. Federal Civilian Executive Branch agencies were required to apply the fixes by June 6, 2026. The article states that patches were released on May 25, 2026, and that affected versions are all versions prior to 1.11.12.

  • A practical detection indicator is a CacheWarmer cookie value matching CacheWarmer:(Tz|Qz|YT), because serialized PHP objects base64-encode to values beginning with Tz, Qz, or YT.
  • The true exposure may be larger than Sansec’s estimate of about 6,000 stores running Mirasvit extensions because CDNs such as Cloudflare can mask installations.
