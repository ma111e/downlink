The FBI is warning about the Kali365 phishing-as-a-service (PhaaS) platform, which targets Microsoft 365 accounts by exploiting the OAuth 2.0 device authorization grant flow. Kali365 first appeared in April 2026 and is distributed via Telegram channels, giving cybercriminals a way to compromise accounts without stealing passwords or intercepting MFA codes. The platform uses device code phishing, a technique that abuses Microsoft’s legitimate device login portal (microsoft.com/devicelogin), designed for devices with limited input capabilities to authenticate via a short code.

In a device code phishing attack, the threat actor initiates the device authorization process to generate a code, then lures the victim into entering that code on the genuine Microsoft login page through phishing or social engineering. Once the victim enters the code and completes the MFA challenge, Microsoft issues an OAuth access token to the attacker, granting full access without requiring the attacker to solve any MFA challenges. This token then provides access to all single-sign-on applications, including Microsoft 365 and Salesforce, enabling data theft.

Kali365 is designed to be accessible to low-skilled attackers, providing AI-generated phishing lures, automated campaign templates, real-time victim-tracking dashboards, and automated token-capture functionality. Arctic Wolf researchers discovered that Kali365 operates as a professional business, with admins managing product development, resellers promoting the service, and affiliates conducting the actual phishing attacks. The platform offers two attack modes: device code phishing and an adversary-in-the-middle mode called “Cookie Link,” which proxies victims through attacker-controlled infrastructure to capture authenticated browser sessions and tokens after the victim logs in and completes MFA.

Arctic Wolf observed a widespread campaign in April 2026 where phishing emails directed victims to the device code login portal. Compromised accounts gave attackers access to mailboxes, where they created hidden inbox rules to mask their activity, and in some cases registered new devices in victims’ environments to prolong access. The attacks were not limited to a single region but targeted organizations worldwide.

The FBI’s public service announcement recommends that organizations restrict or completely block device code authentication flows using Conditional Access policies where possible, audit existing device code usage, and block authentication transfer policies that allow sessions to move between devices. The FBI has issued a public service announcement (PSA 260521) warning about the Kali365 phishing-as-a-service platform, which is used to hijack Microsoft 365 accounts by abusing OAuth device code authentication. Technical Mechanism: Device Code Phishing

Device code phishing exploits the legitimate OAuth 2.0 Device Authorization grant flow, which Microsoft created for devices with limited input capabilities (smart TVs, conference room systems, printers, IoT devices). The flow works as follows:

  • The device generates a short code and displays it to the user.
  • The user visits microsoft.com/devicelogin on another device, enters the code, and authenticates.
  • After successful authentication, the original device receives an access token.

Attackers abuse this flow by initiating the device authorization process themselves to generate a code. They then trick the victim into entering that code on Microsoft’s legitimate login page via phishing emails or social engineering. Because the portal is genuine and uses a valid Microsoft URL, traditional phishing detection based on domain reputation fails.

When the victim enters the code and completes any MFA challenge, Microsoft issues an OAuth access token to the attacker’s device. This token grants the attacker full access to all single-sign-on applications the victim normally uses — such as Microsoft 365, Salesforce, or other SaaS platforms — without the attacker ever needing to solve an MFA prompt. The entire process occurs on legitimate infrastructure, making it extremely difficult to block without restricting the underlying authentication flow itself.

Kali365 Phishing-as-a-Service Platform

Kali365 is designed as a turnkey criminal service with a formal business structure. Arctic Wolf researchers identified three tiers of participants:

  • Admins: Manage product development and platform operations.
  • Resellers: Promote the service to other threat actors.
  • Affiliates: Conduct the actual phishing attacks using the platform.

The platform offers two distinct attack modes:

  1. Device Code Phishing: The primary method described above, where victims are directed to Microsoft’s device login portal.
  2. Adversary-in-the-Middle (AitM) – “Cookie Link”: This mode proxies victims through attacker-controlled infrastructure that captures authenticated browser sessions, session cookies, and tokens after the target logs in and solves any MFA challenges.

Additional capabilities provided to attackers include:

  • AI-generated phishing lures.
  • Automated campaign templates.
  • Real-time victim-tracking dashboards.
  • Token-capture functionality.

Once victims unknowingly authorized the attacker’s device, threat actors gained full inbox access.

Post-compromise activities included:

  • Creation of malicious inbox rules designed to hide attacker activity from the victim.
  • Registration of new devices in victims’ Microsoft environments, further extending persistent access to the breached network.

The article also notes that extortion gangs, including the ShinyHunters cybercrime group, were previously observed targeting Microsoft Entra accounts using device code phishing combined with voice phishing (vishing), indicating that this technique is used by multiple threat actors.

Recommended Defensive Measures

The FBI recommends the following actions to defend against device code phishing attacks:

  • Restrict or completely block device code authentication flows using Conditional Access policies wherever possible.
  • Audit existing device code usage to identify any legitimate dependencies.
  • Block authentication transfer policies that allow authentication sessions to move between devices.
  • Impeded organizations should report incidents to the Internet Crime Complaint Center (IC3) and preserve phishing emails, suspicious login information, and records of unauthorized device registrations.

The article highlights that device code phishing is not unique to Kali365 and has seen widespread adoption in 2026. Other phishing-as-a-service platforms now employing the same technique include:

  • EvilTokens PhaaS – Also targeting Microsoft 365 and Entra accounts through device code phishing.
  • Tycoon2FA – Hijacking Microsoft 365 accounts via the same device code abuse method.

This trend underscores that the device code authentication flow has become a favored attack vector across the cybercriminal ecosystem, making the FBI’s recommendations especially urgent for organizations relying on Microsoft 365 and Entra ID.

Summary
A critical Ghost CMS SQL injection flaw (CVE-2026-26980) was exploited to seize admin API keys and silently inject malicious JavaScript into over 700 legitimate sites, fueling large-scale ClickFix attacks that trick visitors into running malware.

Attackers are exploiting a critical SQL injection vulnerability (CVE-2026-26980, CVSS 9.4) in Ghost CMS to steal admin API keys and silently inject malicious JavaScript into more than 700 legitimate websites. The campaign, first spotted on May 7, 2026, weaponizes the flaw to poison site content and fuel fake CAPTCHA-based ClickFix attacks, leveraging the compromised sites’ trusted reputations across sectors like universities, blockchain, AI, and security research.

The injected JavaScript acts as a two-stage loader that fetches a cloaked payload from an external domain using the commercial Adspect service, ensuring only real visitors receive the attack while scanners see benign content. Ultimately, victims who follow the fake CAPTCHA instructions run a dropper that installs a persistent remote-access trojan (a signed PuTTY or a modified Grape desktop client), giving attackers long-term control of the infected machines.

A large-scale website poisoning campaign is exploiting a recently disclosed Ghost CMS vulnerability to hijack sites and serve ClickFix malware. The flaw, CVE-2026-26980, is a SQL injection in the Content API with a CVSS score of 9.4 that allows unauthenticated attackers to read arbitrary database contents, including the admin API key. The vulnerability was discovered by Anthropic using Claude and patched in Ghost version 6.19.1 (February 2026). QiAnXin XLab, which reported the exploitation, states the campaign started on May 7, 2026, and involves at least two distinct threat clusters.

Attackers use the SQL injection to obtain the target site’s admin API key without authorization, then leverage the Ghost Admin API to bulk-modify articles. They inject a two-stage JavaScript loader at the bottom of pages. This loader contacts an external domain, “clo4shara[.]xyz/11z77u3.php”, which hosts a traffic distribution script powered by Adspect, a commercial cloaking service. The script collects browser fingerprints and uses 19 different commands to selectively redirect, pop up, or download payloads—only real victims receive the malicious content, while crawlers see a benign page.

Visitors to compromised pages encounter a fake CAPTCHA verification challenge embedded in an iframe. If they follow the instructions, they are instructed to copy and paste a Base64-encoded command into the Windows Run dialog. This command acts as a dropper: it unpacks a ZIP archive, runs a batch script that downloads a DLL or JavaScript payload, and executes it via rundll32.exe while opening a decoy webpage. Early iterations used a valid code-signed PuTTY executable as the final payload; later versions switched to a modified open-source Grape desktop client packaged with an Inno Setup installer.

The Grape-based backdoor achieves persistence and contacts “web-telegram[.]ug” every 30 seconds to fetch attacker commands, enabling remote execution of arbitrary JavaScript or binaries. Over 700 sites have been compromised, including those of universities, blockchain platforms, AI firms, SaaS providers, security researchers, media outlets, and fintech companies. Using legitimate, trusted websites dramatically increases the success rate of the ClickFix attacks.

Ghost CMS administrators are urged to upgrade to the latest version, rotate all credentials, purge injected content, audit access logs for suspicious activity, and warn users who may have visited the site during the contamination window. The incident underscores how a single vulnerability in a widely used CMS can enable a stealthy, bulk-compromise operation that blends malicious code into authentic content via legitimate API channels.

Overview

A critical SQL injection vulnerability in Ghost CMS (CVE-2026-26980) is being actively exploited in a large-scale website poisoning campaign. Attackers steal the admin API key from vulnerable instances, then abuse the Ghost Admin API to inject malicious JavaScript into articles, ultimately delivering ClickFix malware to site visitors. QiAnXin XLab first detected the activity on May 7, 2026, and reports that over 700 websites across multiple sectors have been compromised.

Vulnerability Details

  • CVE ID: CVE-2026-26980
  • CVSS Score: 9.4 (Critical)
  • Flaw Type: SQL injection in Ghost’s Content API
  • Impact: Unauthenticated attackers can read arbitrary data from the database, including the admin API key
  • Discovery: Anthropic found the flaw using Claude
  • Patch: Fixed in Ghost version 6.19.1 (February 2026)
  • Exploitation Severity: Access to the admin API key enables direct modification of published articles via the Admin API

Campaign Execution

Initial Compromise

Threat actors exploit the SQL injection flaw to retrieve the site’s Admin API key without any authentication. With this key, they invoke the Ghost Admin API to batch-modify articles, inserting a malicious JavaScript loader at the bottom of each page.

Injected Loader

The injected code is a two-stage loader that contacts the external domain clo4shara[.]xyz/11z77u3.php at runtime. This architecture allows attackers to swap out the final payload without altering the injected code across hundreds of infected sites.

Cloaking and Traffic Distribution

The PHP script at the loader’s destination is a traffic distribution system powered by Adspect, a commercial ad-fraud cloaking service. It collects browser fingerprints and uploads them to the server, then issues one of 19 supported commands, such as redirection, popup generation, or file download. This ensures that only real, unsuspecting users are served the malicious payload, while security scanners and crawlers see only benign content.

ClickFix Delivery Chain

  1. Fake CAPTCHA: Victims on a compromised page see an iframe with a fake CAPTCHA verifying they are human.
  2. Run Command: The CAPTCHA instructs them to copy a Base64-encoded string and paste it into the Windows Run dialog.
  3. Dropper: The command downloads and extracts a ZIP archive, runs a Windows batch script.
  4. Script Actions: The batch script executes PowerShell to download a DLL (or JavaScript) from a remote domain, launches it via rundll32.exe, and opens a decoy webpage as distraction.

Final Payloads and Persistence

Early Payload: Signed PuTTY

When the dropper delivered a DLL, the ultimate payload was a legitimate, code-signed PuTTY client. While the exact role of the PuTTY binary is not detailed, its use of a valid signature aids evasion.

Later Payload: Modified Grape Desktop Client

Later iterations replaced the DLL with a JavaScript payload. This results in an Inno Setup installer that deploys a trojanized version of the open-source Grape desktop client. The modified application:

  • Achieves persistence on the host.
  • Contacts the remote server web-telegram[.]ug every 30 seconds.
  • Receives and executes commands, including running arbitrary JavaScript code or launching executable files.

Infrastructure and Attribution

  • Loader Domain: clo4shara[.]xyz
  • C2 for Backdoor: web-telegram[.]ug
  • Cloaking Service: Adspect (commercial)
  • Threat Clusters: At least two distinct groups are involved, with some sites compromised within a single day.

Impact

  • Scale: Over 700 websites poisoned.
  • Sectors Affected: Universities, blockchain, AI, SaaS, security research, media, and financial technology.
  • Edge Given: Because the sites are legitimate and trusted, visitors are more likely to fall for the fake CAPTCHA, increasing ClickFix success rates.

Recommendations

Ghost CMS operators should immediately:

  • Upgrade to version 6.19.1 or later
  • Rotate all admin API keys and other credentials
  • Scan and clean articles for injected JavaScript
  • Audit access logs for unauthorized Admin API calls
  • Notify users who may have visited the site during the contamination period

This campaign demonstrates how a single vulnerability in a widely used CMS can be weaponized to turn hundreds of trusted websites into malware distribution platforms, using legitimate API features to make detection and cleanup more difficult.

  • The SQL injection flaw directly exposed the admin API key, allowing attackers to modify sites via legitimate API calls, making the injected content indistinguishable from normal admin changes and bypassing typical integrity checks.
  • The attackers relied on a commercial cloaking service (Adspect) that provides 19 distinct commands for fine-grained remote browser control, enabling tailored attacks beyond simple redirection or malware delivery.
  • The campaign targeted security research websites along with other sectors, potentially undermining the credibility of those sites and tricking a more security-savvy audience, thereby amplifying the attack's effectiveness.
  • The payload evolved mid-campaign from a DLL dropper to a JavaScript-based loader, indicating active attacker monitoring and adaptation to evade detection or improve persistence.

    2. Staying Tuned: Networks change due to new applications, cloud workloads, unknown devices, and AI-driven data flows. Regular tuning keeps the baseline calibrated and prevents false positives. AI can help spot emerging patterns before they become noise.
    3. SOC Integration: NDR data can fuel other SOC tools. Better data quality yields cleaner results. When AI has high-fidelity data, it distinguishes true threats from false positives more accurately. Organizations with mature deployments use APIs and detection feeds strategically, letting the NDR AI handle correlation before alerts reach other platforms, thus reducing noise before it hits the analyst queue.

02 Active Exploitation of Critical Vulnerabilities: Drupal, Ghost CMS, Linux Kernel, and Microsoft Defender Zero-Days

A cluster of critical flaws is under active attack. Drupal released emergency patches for CVE-2026-9082, a critical SQL injection vulnerability affecting sites using PostgreSQL, with exploitation already observed in the wild. The Ghost CMS platform was hit by CVE-2026-26980 (CVSS 9.4), an unauthenticated SQL injection that allows retrieval of admin API keys. Threat actors used this access to bulk-inject JavaScript loaders into over 700 legitimate websites spanning universities, blockchain, AI, SaaS, media, and fintech sectors. The injected code delivers a cloaked second-stage payload that presents victims with a fake CAPTCHA as part of a ClickFix attack, ultimately dropping a signed PuTTY or modified Grape desktop client for persistence and remote control.

On the infrastructure side, a nine-year-old local privilege escalation in the Linux kernel (CVE-2026-46333) enables unprivileged users to execute commands as root on default installations of Debian, Fedora, and Ubuntu. Microsoft disclosed that two Defender vulnerabilities—one allowing SYSTEM-level privilege escalation and the other causing denial-of-service—are also being actively exploited. Ransomware accounted for 48% of breaches, though the median ransom payment declined to nearly $140,000.

It equips low-skilled actors with AI-generated lures, real-time dashboards, and automated token capture, reflecting the commoditization of sophisticated phishing techniques. Check Point Research reported a campaign in which a single operator compromised nine Mexican government agencies and executed over 5,000 automated commands, illustrating that AI-driven operations are now in routine criminal use.

Anthropic’s restricted Claude Mythos model, which possesses advanced code reasoning and can autonomously develop functional cyberattacks at a professional level, briefly surfaced in Claude Code and Claude Security, suggesting a public rollout is approaching.
