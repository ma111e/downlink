Overview

Socket detected a coordinated PyPI compromise involving 37 malicious wheel artifacts across 19 packages. The compromised releases shipped a *-setup.pth file that attempts to execute automatically during Python startup, download the Bun JavaScript runtime, and run an obfuscated JavaScript payload named _index.js.

Socket identified the malicious package cluster minutes after publication using its AI malware detection system. The company attributes the tradecraft to the Shai-Hulud / Miasma lineage and describes this Hades-themed PyPI wave as a branch of the same Mini Shai-Hulud / Miasma campaign rather than as a standalone Python malware incident.

Campaign Context

The article says the campaign is cross-runtime: Python packages act as the delivery vehicle, but the payload runs under Bun as a heavily obfuscated JavaScript stealer. Socket identifies Bun staging as a key fingerprint of this malware family because Shai-Hulud-style payloads download and install Bun rather than assuming Node.js, Python, or another local runtime is available.

The campaign marker changed from earlier Shai-Hulud / Miasma waves. Earlier reporting tied a Red Hat Cloud Services wave to the Zelda-themed marker Miasma: The Spreading Blight, while this PyPI wave uses Hades-themed GitHub exfiltration markers. Recovered markers include the repository description Hades - The End for the Damned and generated repository-name components such as stygian, tartarean, cerberus, charon, styx, lethe, thanatos, and persephone.

Socket says it is tracking 448 affected artifacts across npm and PyPI in the broader campaign. This includes 411 npm artifacts across 106 packages and 37 malicious PyPI wheels across 19 projects. At the time of writing, PyPI had already quarantined a number of affected releases, and Socket reported the remaining ones to the PyPI security team.

Affected PyPI Packages

The 37 compromised PyPI artifacts span 19 packages and appear to come from a single maintainer-account takeover. The article says consecutive patch releases were mass-published across the author’s portfolio at once.

Socket says the risk is concentrated in established bioinformatics tools, including:

  • dynamo-release, associated with the aristoteleo/dynamo single-cell RNA-velocity and expression-dynamics framework.
  • spateo-release, described as a spatial-transcriptomics sibling.
  • coolbox, described as GangCaoLab’s Jupyter-based multi-omics genomic visualization toolkit for Hi-C, ChIP-Seq, and RNA-Seq tracks.
  • ufish and napari-ufish, described as deep-learning FISH spot-detection tools.

The article says these tools are real and widely used in the research community, with cumulative download totals in the low-to-mid hundreds of thousands. It describes the remaining affected packages as lower-traffic agent, task-execution, function-description, and lab-utility libraries that were caught in the same blast rather than independently valuable targets.

Malicious Wheel Structure

Socket observed a simple and suspicious pattern in the malicious wheels:

  • A JavaScript payload named _index.js.
  • A *-setup.pth file.

The *-setup.pth file contains a single executable Python line. Python’s site module processes .pth files during interpreter startup, and lines beginning with import followed by a space or tab are executed. This means the attacker can attempt startup execution after installation without requiring the victim to import the compromised package.

Loader Behavior

The loader attempts to perform the following actions:

  • Create a sentinel file at tempfile.gettempdir()/.bun_ran.
  • Locate _index.js next to the package or one subdirectory below it.
  • Download Bun v1.3.13 from GitHub if no cached Bun binary exists under the temp directory.
  • Run bun run _index.js.
  • Write the sentinel to avoid repeated execution.

Socket highlights an implementation caveat for defenders. In standard CPython, executable .pth lines are processed by the site module, and __file__ can resolve to site.py rather than to the .pth file. Socket says a local CPython reproduction of this loader shape did not automatically resolve the adjacent package _index.js through os.path.dirname(__file__), so exploitability should be validated per artifact even though the wheels remain malicious.

Why the .pth Mechanism Matters

The article explains that .pth files are a legitimate Python startup feature used to add paths to sys.path and support import hooks. Python also supports executable lines beginning with import, and those lines can run at every Python startup.

Socket says this turns a compromised wheel into a delayed execution trigger. The next python, pip, test run, notebook kernel, CI job, or package-management command that starts Python may process the malicious .pth file.

The article compares this to npm install hooks used in Shai-Hulud and Miasma activity. The syntax differs, but the security consequence is similar: dependency installation creates an execution edge before application code is reviewed or invoked.

Payload Deobfuscation

Socket’s static deobfuscation of _index.js recovered multiple layers:

  1. An outer JavaScript wrapper using try { eval(...) }, a large character-code array, and a ROT-style alphabet substitution.
  2. An AES-GCM loader that imports node:crypto, decrypts two AES-128-GCM blobs, writes the main payload to a random /tmp/p*.js file, and runs it with Bun.
  3. A Bun bootstrapper that downloads Bun from https://github.com/oven-sh/bun/releases/download/bun-v1.3.13/.
  4. A main JavaScript payload using a rotated string table, a custom PBKDF2/SHA256-based string decoder, and an additional AES-256-GCM plus gzip string layer.

The submitted _index.js sample begins with a try{eval(...)} wrapper and a long character-code array, matching the obfuscated first stage Socket recovered during static analysis.

  • .pth executable import line.
  • Remote runtime or executable download.
  • Temp-directory binary installation.
  • Subprocess execution.
  • _index.js or JavaScript payload handoff.

  • _index.js
