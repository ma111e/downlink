Vulnerability Overview

Cybersecurity researchers have disclosed a series of security flaws affecting NGINX Plus and NGINX Open Source, most notably a critical heap buffer overflow that went undetected for 18 years. The primary vulnerability, identified as CVE-2026-42945 and codenamed NGINX Rift, resides within the ngx_http_rewrite_module. It carries a CVSS v4 score of 9.2 and poses a severe risk to unauthenticated remote attackers.

Exploitation Mechanics and Impact

The flaw triggers when a rewrite directive is followed by a rewrite, if, or set directive that employs an unnamed PCRE capture, such as \(1 or \)2. If the replacement string for this capture includes a question mark, sending a crafted HTTP request causes a heap buffer overflow in the NGINX worker process. This overflow can reliably crash the worker, resulting in a denial of service. On systems where Address Space Layout Randomization is disabled, the attacker can achieve remote code execution. The corruption bytes are directly derived from the attacker’s URI, allowing precise control over the heap corruption.

Disclosure and Patching Status

The vulnerability was responsibly disclosed to F5 on April 21, 2026, and addressed in the following product versions:

  • NGINX Plus R32 through R36, with fixes introduced in R32 P6 and R36 P4.
  • NGINX Open Source 1.0.0 through 1.30.0, patched in versions 1.30.1 and 1.31.0.
  • NGINX Open Source 0.6.27 through 0.9.7, which will not receive updates.
  • NGINX Instance Manager 2.16.0 through 2.21.1.
  • F5 WAF for NGINX 5.9.0 through 5.12.1.

  • NGINX Ingress Controller 3.5.0 through 3.7.2, 4.0.0 through 4.0.1, and 5.0.0 through 5.4.1.

Mitigation Recommendations

F5 advises applying the latest patches for optimal security. If immediate patching is not feasible for CVE-2026-42945, administrators should modify their rewrite configurations. The recommended workaround involves replacing all unnamed regex captures with named captures in every affected rewrite directive to prevent the heap overflow condition.
