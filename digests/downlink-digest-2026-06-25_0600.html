Mass npm Supply Chain Attack: 20 Leo Platform Packages Compromised
stepsecurity.io · 25 Jun 2026 08:52 UTC openai-codex / gpt-5.5
TAGSbinding-gypbunbypass-2fagithub-actionsleo-platformmiasmanode-gypnpmphantom-gypstepsecuritysupply-chain-worm
94
In plain words

An attacker broke into the publishing access for 20 Leo Platform software packages on npm and released harmful new versions all at once. People and companies that installed those versions could have had private keys, cloud logins, GitHub tokens, package publishing access, and password manager data stolen during installation. This matters because the stolen access could let the attacker break into company systems and even publish more poisoned software packages under the victim’s name, spreading the problem further.

Summary
Twenty Leo Platform npm packages were compromised in a coordinated supply-chain attack that used a stealthy binding.gyp install hook to steal CI/CD, cloud, registry, and password-manager secrets from GitHub Actions environments. The campaign appears tied to the earlier Miasma operation and can also propagate by abusing stolen npm tokens to publish more malicious packages.

On June 24, 2026, an attacker published malicious versions of 20 npm packages in the Leo Platform ecosystem in a coordinated burst lasting less than three seconds. The packages together receive about 13,600 downloads per week. The article says the attacker gained access to Leo Platform maintainer credentials and published versions including leo-logger@1.0.8, leo-sdk@6.0.19, leo-aws@2.0.4, and others.

The malicious packages contain an identical CI/CD attack toolkit that steals secrets from GitHub Actions runners, cloud credential stores, package registries, and password managers. The payload exfiltrates stolen credentials through GitHub using the victim’s own token, rather than contacting an external attacker-controlled domain. It can also use an npm token to publish malicious versions of packages the victim can access through the bypass_2fa mechanism.

The article links the Leo Platform campaign to the earlier Miasma campaign because both use the same Phantom Gyp binding.gyp install hook, the same three-layer obfuscation chain, and the same Bun v1.3.13 download URL. The payload can read /proc/{pid}/mem from the GitHub Actions Runner.Worker process to recover secrets masked from workflow logs. The report lists indicators of compromise including the shared publish timestamp, binding.gyp hook syntax, Bun download path, temporary /tmp/p*.js execution, and SHA1 hashes for the malicious package versions.

  • A package can execute malicious code during installation through a binding.gyp file even when package.json has no install script, so the article recommends treating packages with binding.gyp but no C++ sources or .node output as suspicious.
  • The payload avoids using an attacker-controlled server for stolen data by committing encrypted secrets to GitHub repositories through the victim’s own GitHub token, making outbound-domain based detection less reliable.
  • The simultaneous three-second publish window across all 20 packages is an indicator that defenders can use to identify affected Leo Platform versions from this campaign.
  • A cooldown period for newly published npm package versions would have prevented immediate exposure to these malicious releases because all affected versions were newly published in a coordinated burst.
  • The article identifies reading /proc/{pid}/mem for the GitHub Actions Runner.Worker process as a definitive malicious behavior in CI/CD environments.
Key Points
  • 01On June 24, 2026, an attacker published malicious versions of 20 npm packages in the Leo Platform ecosystem within a window of less than three seconds.
  • 02The compromised packages contain a CI/CD attack toolkit that steals secrets from GitHub Actions runners, cloud credential stores, package registries, and password managers, then exfiltrates them using the victim’s own GitHub token.
  • 03The Leo Platform attack uses the same toolkit and delivery mechanism documented in the earlier Miasma campaign, including a Phantom Gyp binding.gyp install hook, a three-layer obfuscation chain, and Bun runtime evasion.
  • 04The payload can locate the GitHub Actions Runner.Worker process, read process memory to recover masked secrets, sweep credentials from AWS, GCP, Azure, HashiCorp Vault, Kubernetes, npm, PyPI, RubyGems, JFrog, GitHub PATs, and 1Password, and modify GitHub Actions workflow files.
  • 05The affected malicious versions include leo-logger@1.0.8, leo-sdk@6.0.19, leo-aws@2.0.4, leo-config@1.1.1, leo-streams@2.0.1, serverless-leo@3.0.14, and 14 other Leo Platform packages listed in the report.

New Mistic Backdoor Linked to KongTuke in ClickFix and ModeloRAT Campaigns
thehackernews.com · 25 Jun 2026 08:54 UTC openai-codex / gpt-5.5

In plain words

A group called KongTuke has been linked to a new hidden tool named Mistic that lets attackers quietly control infected computers, move files, and erase itself, while also using another tool called ModeloRAT. Since April 2026, the activity has targeted organizations in insurance, education, IT, and professional services, often tricking people through fake browser or support messages into running commands. This matters because the attackers appear to be looking for access they can sell to others, including ransomware operators, which can lead to wider break-ins and extortion.

Summary
A new stealthy in-memory backdoor called Mistic has been linked to the KongTuke initial access broker and is being delivered through ClickFix-style campaigns alongside ModeloRAT. The activity appears aimed at gaining low-visibility access to organizations that can be sold to ransomware affiliates, including those tied to Qilin.

A new stealthy backdoor named Mistic, also tracked as MLTBackdoor, has been deployed in suspected financially motivated attacks since April 2026. The activity has targeted organizations in the insurance, education, IT, and professional services sectors. Symantec and Carbon Black’s Threat Hunter Team linked the backdoor to the KongTuke initial access broker and said it has been dropped alongside ModeloRAT, a Python remote access trojan previously attributed to the group.

Mistic is delivered through ClickFix-related activity, a tactic that tricks victims into running commands under false pretenses. ModeloRAT was previously observed in CrashFix activity involving a malicious Chrome extension that intentionally crashed a browser and prompted victims to run commands, and in another campaign that used DNS lookups to retrieve the next payload. Zscaler ThreatLabz also highlighted Mistic’s use of ClickFix as a delivery vector and tied the activity to a ransomware-related actor seeking a foothold for lateral movement.

Broadcom reported that Mistic uses DLL side-loading with trusted Microsoft endpoint security tooling, specifically MpExtMs.exe, to blend in and avoid raising red flags. The backdoor runs payloads in memory, can upload and download files, move or delete files, create folders, change how often it checks a remote server for commands, execute code in memory, load Beacon Object Files, and terminate and delete itself. Broadcom said the targeting appears opportunistic and noted that ModeloRAT has been observed in attacks that deployed Qilin ransomware.

  • The use of trusted Microsoft endpoint security tooling, specifically MpExtMs.exe, for DLL side-loading means Mistic is designed to blend into environments by abusing software that defenders may normally trust.
  • KongTuke’s delivery methods have shifted across compromised WordPress traffic distribution, malicious Chrome extensions, DNS-based ClickFix staging, and fake Microsoft Teams IT Support messages, showing that the same access broker is rapidly changing lures while continuing to deliver related tools.
  • Mistic’s ability to execute code in memory and load Beacon Object Files allows operators to expand capabilities without writing additional tools to disk, reducing the forensic evidence available after an intrusion.
  • Broadcom’s assessment that access may be sold after opportunistic targeting makes the affected sectors broader than a single industry campaign, with insurance, education, IT, and professional services all exposed to downstream ransomware activity.
Key Points
  • 01A new stealthy backdoor named Mistic, also tracked as MLTBackdoor, has been deployed in suspected financially motivated attacks against organizations in the insurance, education, IT, and professional services sectors since April 2026.
  • 02Symantec and Carbon Black linked Mistic to the KongTuke initial access broker and reported that it has been dropped alongside ModeloRAT, a Python remote access trojan previously attributed to the group.
  • 03Mistic uses DLL side-loading with the trusted Microsoft endpoint security tool MpExtMs.exe, runs payloads directly in memory, and includes a kill switch that can terminate and delete itself.
  • 04The backdoor can upload, download, move, rename, or delete files, create folders, change its command polling interval, execute code from its command-and-control server in memory, and load Beacon Object Files.
  • 05Broadcom said the targeting appears opportunistic and noted that ModeloRAT has been observed in attacks that deployed Qilin ransomware.
