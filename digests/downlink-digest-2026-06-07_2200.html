Overview

Microsoft has created an open-source fork of Windows Terminal called Intelligent Terminal. The article describes it as a version of Terminal that lets users use AI directly inside the terminal without interfering with the regular session.

Microsoft describes Intelligent Terminal as a built-in assistant that can help users explain errors, draft commands, and fix problems without leaving the terminal. The app is presented as a separate application rather than a feature currently included with Windows installations.

Core Functionality

Intelligent Terminal is designed to integrate an AI assistant into the terminal workflow. According to the article, the agent can stay aware of what is happening in the terminal and help when a command fails.

The app can also remember active and past agent sessions. This allows users to return to earlier agent work without losing their place.

The article states that Intelligent Terminal can help with several terminal-related tasks, including:

  • Explaining errors.
  • Drafting commands.
  • Helping fix problems.
  • Assisting when a command fails.
  • Reopening previous agent work through session management.

First-Run Configuration

When Intelligent Terminal is opened for the first time, it lets the user choose the AI agent for the Terminal pane. In the article’s example screenshot, the listed options are:

  • GitHub Copilot.
  • Claude.
  • Codex.
  • Gemini.

The article notes that GitHub Copilot is shown as “will be installed,” while Claude, Codex, and Gemini are shown as already installed.

Error Detection and Error Suggestions

Intelligent Terminal includes separate toggles for Automatic error detection and Automatic error suggestion.

The article explains that Automatic error detection allows Terminal to notice failed commands. Automatic error suggestion goes further by sending the error to the selected AI agent for a possible fix.

This distinction is important in the article’s description because detection identifies failed commands, while suggestion involves asking the selected AI agent to provide a possible remediation.

Session Management

The article identifies Session management as one of Intelligent Terminal’s most notable features. This option lets the app track active and past agent sessions.

The author specifically highlights the ability to resume sessions as one of Intelligent Terminal’s best features. The article contrasts this with the current Windows Terminal option that can reopen previously closed tabs but does not restore previous sessions.

The author also notes that while Claude Code has a built-in resume skill, they believe it often makes the model perform worse. Intelligent Terminal is presented as addressing this issue by allowing users to move back and forth between earlier agent work.

User Interface and Hands-On Experience

After configuration, Intelligent Terminal opens with an AI pane below the shell. The pane displays the message “Welcome to Intelligent Terminal.”

In the author’s hands-on example, Claude was selected as the Terminal AI model. As a result, Claude Code ran inside the AI pane.

The article describes Claude Code as being able to plan a coding task and then ask the user whether they wanted to:

  • Auto-accept edits.
  • Manually approve edits.
  • Keep planning.

The interface also includes controls on both sides of the terminal. On the left side, users can show or hide the agent panel and toggle error detection through an icon. On the right side, an agent management icon opens the session management panel and agent status bar.

Availability and Deployment Approach

The article states that Terminal AI is a strong idea but is not intended for everyone. Microsoft has therefore made Intelligent Terminal a separate app.

The article explicitly notes that Intelligent Terminal is not included with Windows installations yet. Users who are interested can download it from the Microsoft Store or GitHub.
