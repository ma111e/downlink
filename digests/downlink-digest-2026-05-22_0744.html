01 Developer and CI/CD Supply Chain Compromise

A new Shai-Hulud and Mini Shai-Hulud wave affected the npm ecosystem at scale, with researchers reporting 639 malicious versions across 323 packages, heavily concentrated in Alibaba’s AntV ecosystem and packages tied to the compromised npm maintainer account atool. The malware targeted developer and CI/CD secrets, including GitHub, cloud, npm, SSH, Kubernetes, Vault, Stripe, and database credentials, and used mechanisms such as install-time hooks, poisoned GitHub commits, GitHub API exfiltration, Session P2P, and the C2 domain t.m-kosche.com.

Several reports described the campaign’s ability to weaponize trusted automation paths. One variant read GitHub Actions runner process memory to recover masked secrets, while another abused OIDC tokens from compromised CI environments to generate valid Sigstore provenance attestations, making malicious npm packages appear legitimately signed. Related GitHub Actions compromise activity included actions-cool/issues-helper and actions-cool/maintain-one-comment, where tags were redirected to malicious commits that harvested CI/CD credentials.

Microsoft’s durabletask Python SDK on PyPI was also compromised through malicious versions 1.4.1, 1.4.2, and 1.4.3 uploaded directly to PyPI on May 19. The injected Linux import-time payload downloaded rope.pyz, which stole cloud secrets, installed persistence, and attempted lateral movement through AWS SSM and Kubernetes. Separately, Nx Console version 18.95.0 in the VS Code Marketplace was compromised after a developer machine compromise exposed GitHub credentials, enabling a malicious orphan commit in the official nrwl/nx repository to deliver a multi-stage credential stealer.

02 Microsoft Trust, Identity, and Cloud Abuse

Microsoft disrupted Fox Tempest, a malware-signing-as-a-service operation that abused Microsoft Artifact Signing to create fraudulent code-signing certificates for cybercriminal customers. Microsoft said the service created more than 1,000 certificates and hundreds of Azure tenants and subscriptions, and that signed malware was used in campaigns involving Oyster, Lumma Stealer, Vidar, Rhysida, Akira, INC, Qilin, and BlackByte. The operation allegedly helped malware impersonate legitimate software such as Microsoft Teams, AnyDesk, PuTTY, and Webex.

Microsoft also detailed Storm-2949 activity targeting Microsoft 365 and Azure production environments through social engineering and abuse of Entra ID Self-Service Password Reset. The actor used privileged credentials to reset passwords, remove MFA controls, enroll attacker-controlled authenticators, enumerate environments through Microsoft Graph, and access OneDrive, SharePoint, virtual machines, storage accounts, key vaults, app services, and SQL databases.

OAuth consent phishing remained a major identity abuse theme. Reporting on EvilTokens described more than 340 Microsoft 365 organizations compromised after users entered device login codes, completed MFA on legitimate Microsoft infrastructure, and granted OAuth access that produced refreshable tokens. The activity highlights how MFA completion can coexist with durable attacker access when users authorize malicious or deceptive applications.

03 Critical Vulnerabilities in AI, Linux, Network, and Enterprise Software

CVE-2026-45829, a max-severity flaw in the Python FastAPI version of ChromaDB, allows unauthenticated arbitrary code execution on exposed servers by causing ChromaDB to load and execute a malicious Hugging Face model before authentication checks. The issue affects the PyPI package when the Python API server is reachable over HTTP, while local-only deployments and Rust front-end users were reported as unaffected.

Proof-of-concept code was released for DirtyDecrypt, CVE-2026-31635, a Linux kernel local privilege escalation vulnerability tied to a missing copy-on-write guard in rxgk_decrypt_skb(). Reports said affected distributions include those with CONFIG_RXGK enabled, including Fedora, Arch Linux, and openSUSE Tumbleweed. Cisco Talos also disclosed vulnerabilities affecting TP-Link Archer AX53, Adobe Photoshop, OpenVPN, and Norton VPN, including command injection, buffer overflow, denial-of-service, and privilege escalation issues.
