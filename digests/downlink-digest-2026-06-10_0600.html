01 Windows Zero-Days and Public Exploit Pressure

Microsoft’s June 2026 Patch Tuesday addressed three zero-day vulnerabilities known as GreenPlasma, MiniPlasma, and YellowKey. GreenPlasma and MiniPlasma are local privilege escalation flaws in the Collaborative Translation Framework and Cloud Files Mini Filter Driver, respectively, and can allow attackers to obtain SYSTEM privileges on fully patched Windows systems prior to remediation.

YellowKey is a Windows Recovery Environment backdoor affecting unpatched Windows 11 and Windows Server 2022⁄2025 systems, enabling attackers with physical access to bypass BitLocker protections. The vulnerabilities were disclosed by a researcher using the handle Nightmare Eclipse, who framed the release as a protest against Microsoft Security Response Center disclosure practices.

The same researcher has reportedly released proof-of-concept exploits for additional Windows zero-days, including BlueHammer, RedSun, UnDefend, and RoguePlanet. This creates a broader exposure pattern in which defenders face not only patched vulnerabilities, but also an expanding set of public exploit artifacts tied to unresolved or recently disclosed Windows flaws.

02 Critical Enterprise and Internet-Facing Platform Flaws

Aikido reported a critical phpBB authentication bypass vulnerability that can lead to remote code execution and complete takeover of the underlying system. The flaw affects phpBB 3.3.16 and below, as well as 4.0.0-a2, and is exploitable in the default configuration with a single unauthenticated HTTP request to obtain a valid session as any user.

phpBB released version 3.3.17 after the issue was reported through HackerOne on June 2, with a working patch published on June 6. The update also introduces a breaking change for OAuth authentication users because the redirect URI handler has moved to /user/oauth/authenticate/…, which may affect deployments relying on existing OAuth flows.

Ivanti also disclosed two critical Sentry secure mobile gateway vulnerabilities. CVE-2026-10520 is a maximum-severity OS command injection flaw allowing remote code execution as root, while CVE-2026-10523 is an authentication bypass that can allow unauthenticated remote attackers to create rogue administrative accounts and gain full administrative access; Ivanti stated it had no evidence of exploitation at disclosure time.

03 Confirmed Unauthorized Access in ServiceNow Instances

ServiceNow reported that unknown threat actors exploited a security issue to gain greater unauthorized access to susceptible customer instances than intended. The issue does not currently have a CVE identifier, and details first emerged publicly through Reddit before ServiceNow described anomalous activity and successful queries of instance tables affecting a subset of customers.

ServiceNow applied a hosted-instance security update on June 5, changing an endpoint configuration to restrict access to authenticated users. The affected population includes customers on the Australia platform release and customers that made certain configuration changes to instances running releases prior to Australia.

The incident is notable because it involves exploitation against a major enterprise workflow platform and successful access to customer instance data. ServiceNow said affected customers were notified after evidence of successful table queries was observed.
