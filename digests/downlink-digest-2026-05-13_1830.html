Technical Analysis of YellowKey

YellowKey functions as a BitLocker bypass that impacts Windows 11 and Windows Server 2022 and 2025. The exploitation process involves placing specially crafted FsTx files on an external USB drive or the target drive’s EFI partition. After rebooting into the Windows Recovery Environment, attackers hold down the CTRL key to trigger a command shell. This shell gains unrestricted access to the storage volume protected by BitLocker.

Independent analysis confirms that YellowKey exploits NTFS transactions combined with the Windows Recovery image. Specifically, Windows searches for FsTx directories on attached drives and replays NTFS logs, which deletes the winpeshl.ini file. Consequently, the system launches CMD.EXE instead of the actual recovery environment while the disk remains unlocked. The researcher stated that the current public exploit only targets TPM-only auto-unlock configurations and cannot decrypt physically stolen drives. However, the researcher also claimed the vulnerability remains exploitable in TPM plus PIN environments, though they have not released a proof-of-concept for that scenario.

Technical Analysis of GreenPlasma

GreenPlasma is classified as a Windows CTFMON arbitrary section creation elevation of privileges vulnerability. This flaw allows an unprivileged user to create arbitrary memory-section objects within directory objects that are writable by SYSTEM. These created objects can potentially manipulate privileged services or drivers that trust specific paths inaccessible to standard users.

The leaked proof-of-concept is currently incomplete and lacks the necessary component to achieve a full SYSTEM shell. Despite this limitation, the researcher noted that the vulnerability could be adapted to achieve complete privilege escalation.
