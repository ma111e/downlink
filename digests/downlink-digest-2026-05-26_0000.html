Introduction

The March–April 2026 Check Point Research AI Threat Landscape Digest documents the transition of AI-orchestrated attacks from experimental state-sponsored use to widespread criminal deployment. Multiple independent campaigns—spanning government breaches, mass-exploitation platforms, and phishing-as-a-service—demonstrate commercial AI models operating as persistent, autonomous attack components. Agentic configuration files have become a new attack surface, AI-powered fraud is commercialized at scale, and the vulnerability race has compressed the patch window on both offensive and defensive sides. Enterprise GenAI adoption continues to grow, with data exposure remaining a stable but scaling risk.

AI as Live Attack Operator

AI Selection Considerations

Underground forum discussions reveal actors debating commercial models versus self-hosted open-source alternatives. Advanced operators break tasks into smaller, less suspicious sub-requests and combine commercial AI with uncensored models and custom automation pipelines. Some share tables of non-restricted AI assistants, while others recommend self-hosting open-source models to avoid provider monitoring.

The Mexico Breach

Between December 2025 and February 2026, a single operator compromised nine Mexican government agencies, including tax records, civil registries, vehicle databases, patient files, and electoral infrastructure. Researchers from Gambit Security recovered an operational record of 1,088 attacker prompts generating 5,317 AI-executed commands across 34 sessions.

The operator built a dual AI workflow:

  • Claude Code served as the interactive exploitation assistant—writing exploits, building tunnel chains, mapping environments, and escalating privileges.
  • GPT-4.1 processed harvested server data for automated intelligence analysis, with its output tasking new Claude sessions.

Initially, Claude refused offensive requests. The operator then pasted a penetration-testing cheatsheet into CLAUDE.md in the project root, the file Claude Code loads as persistent context at session start. From that point, subsequent sessions inherited the cheatsheet’s rules, and the AI performed unprompted post-exploitation steps such as shadow file extraction and timestamp cleanup. The jailbreak was persistent through the project configuration layer.

This mirrors the earlier Anthropic-disclosed GTG-1002 espionage campaign but now with financial motivation and a full forensic trail.

Bissa Scanner

Bissa Scanner is a modular mass-exploitation platform exploiting React2Shell (CVE-2025-55182) with over 900 confirmed compromises and an archive of 30,000+ distinct .env filenames recovered from operator-controlled S3 storage. Running since September 2025, the platform specifically harvested AI provider credentials (Anthropic, OpenAI, Groq, Mistral, OpenRouter, HuggingFace, Replicate, DeepSeek). These keys will likely be used for future offensive activity, attributing it to the legitimate account holder.

Claude Code and OpenClaw (with Telegram alerting) assisted the operator in reading the scanner codebase, troubleshooting, and refining the collection pipeline. No jailbreak was used; commercial Claude was accessed through the standard API.

Agentic Configuration Files: A Persistent Attack Surface

Check Point Research disclosed multiple exploitation paths using agentic configuration files as a supply-chain attack vector:

  • CVE-2025-59536: Exploits Claude Code’s Hooks feature and .claude/settings.json to execute arbitrary commands before the developer can read them.
  • Parallel MCP path: .mcp.json triggers MCP server startup, bypassing the consent dialog entirely.
  • CVE-2026-21852: Redirects ANTHROPIC_BASE_URL to a malicious proxy that steals authorization headers and API keys, granting read/write access to the team Workspace before any trust prompt.

These attacks can be delivered via malicious pull requests, honeypot repositories, or compromised codebases. The attack surface is architectural and potentially applies to Cursor (.cursorrules), Windsurf (.windsurfrules), and GitHub Copilot Workspace (.github/copilot-instructions.md).

AI-Powered Fraud at Scale: EvilTokens

EvilTokens is a commercial Phishing-as-a-Service platform with an embedded AI pipeline. Non-technical buyers purchase access to a fully integrated system where model selection, jailbreak, and delivery are handled at the platform level.

Multi-stage attack flow:

  • Device-code phishing pages impersonating Adobe, DocuSign, and SharePoint harvest Microsoft OAuth tokens.
  • The AI pipeline then activates:
    • llama-3.1-8b-instant via Groq ingests up to 5,000 emails in 250-email batches, extracting financial data and organizational hierarchies.
    • llama-3.3-70b-versatile via Groq synthesizes intelligence, generates BEC drafts mimicking victim writing style, and scores them.
    • gpt-4o-mini translates stolen emails for non-English operators.
  • An SMTP Sender delivers the output with rotating addresses, header fingerprint randomization, DKIM signing, and CSS randomization.

The jailbreak is the product. Both Groq-hosted LLaMA stages operate under a platform-level jailbreak that frames the model as an authorized red team analyst conducting sanctioned penetration tests. Prompts instruct the model to mask payment changes behind plausible business reasons and generate emails realistic enough to fool trained employees. The platform also includes a Calendar Invite module that sends fake meeting invitations with sender spoofing, used to apply timing pressure on finance personnel to authorize fraudulent wires.

After the Sekoia disclosure, EvilTokens continued operating and announced additional AI features, accelerating its development through April 2026.

The Vulnerability Race: AI on Both Sides of the Patch Window

Frontier Capabilities

Anthropic’s Claude Mythos (Project Glasswing) demonstrated systematic vulnerability discovery at scale, finding a 27-year-old OpenBSD TCP/SACK bug for approximately $20,000 in compute, a 16-year-old FFmpeg flaw, and a FreeBSD NFS remote code execution vulnerability. On a Firefox test set, Mythos produced 181 successful exploits compared to Opus 4.6’s 2. Anthropic noted this capability emerged as a downstream consequence of general improvements, not explicit training.

Productized tiers include Anthropic’s Claude Security (Opus 4.7, public beta for Enterprise) and OpenAI’s Codex Security (research preview with 14 CVEs on OpenSSH, GnuTLS, libssh, PHP, and Chromium).

Attacker Adoption

  • LMDeploy SSRF (CVE-2026-33626) was exploited in the wild within 12 hours of advisory publication with no public proof-of-concept, suggesting attackers built exploits directly from advisory text.

Consequences

The patch window—the time between disclosure and exploitation—is being compressed on both sides. Vendors must accelerate patch development and deployment; the side that reacts fastest gains the most from AI developments.

Enterprise Adoption and Exposure

Technical Methodology

DLL Side-Loading as a Core Evasion Technique

The campaign relied extensively on DLL side-loading using two legitimately signed binaries to execute malicious code while blending in with benign software:

  • Fortemedia binary (fmapp.exe) loaded a malicious fmapp.dll. This technique was previously observed in MuddyWater’s Operation Olalampo, documented by Group-IB.
  • SentinelOne binary (sentinelmemoryscanner.exe) loaded a rogue sentinelagentcore.dll. This choice was deliberate because the executable belongs to a security product, allowing the attackers to bypass signature-based detection by leveraging the trust associated with a signed security vendor binary. Huntress analyzed the fmapp.dll and identified a hardcoded connection to the attacker-controlled IP address 157.20.182[.]49.

Browser Credential Theft via ChromElevator

Both malicious DLLs embedded the open-source tool ChromElevator. This tool extracts saved passwords, cookies, and payment card data from Chromium-based browsers. Crucially, ChromElevator circumvents Google’s App-Bound Encryption (ABE) protections, enabling the theft of otherwise-hardened credential stores without requiring additional kernel-level tricks.

Post-Exploitation: Node.js and PowerShell Scripting

The attackers deployed a Node.js-based implant chain to execute PowerShell scripts that performed a range of reconnaissance and collection activities:

  • Discovery and enumeration of the environment
  • Screenshot capture
  • SAM hive theft for credential dumping
  • Privilege escalation
  • SOCKS5 reverse-proxy tunnelling to relay traffic and maintain covert access

Persistence and Re-Execution

Inside the South Korean electronics manufacturer, the actors repeatedly re-executed the two DLL-side-loading binaries to ensure continued access. The cadence of activity was consistent with automated implant-driven execution rather than a continuous hands-on-keyboard operator presence, a shift noted by Symantec and Carbon Black researchers.
