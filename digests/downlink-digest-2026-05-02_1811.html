Attack Evolution and Core Mechanism

ConsentFix v3 is an advanced, automated phishing technique designed to hijack Microsoft Azure accounts through OAuth abuse. This iteration improves upon earlier versions by replacing manual copy-paste or drag-and-drop interactions with fully automated workflows. The core mechanism exploits the OAuth2 authorization code flow by targeting first-party Microsoft applications that are pre-trusted and pre-consented within the environment.

Initial Reconnaissance and Infrastructure Setup

The attack sequence initiates with verification of Azure tenant presence by checking for valid tenant IDs. Attackers then harvest employee names, roles, and email addresses to facilitate highly targeted impersonation. To support the operation, multiple accounts are created across a range of services including Outlook, Tutanota, Cloudflare, DocSend, Hunter.io, and Pipedream. These services collectively handle phishing delivery, hosting, data gathering, and exfiltration.

Automation and Token Exfiltration

Pipedream functions as the central automation engine for the attack, fulfilling three critical roles simultaneously:

  • It acts as the webhook endpoint that receives the victim’s authorization code.
  • The platform operates as an automation engine that immediately exchanges the captured code for a refresh token via Microsoft’s API.
  • It serves as a real-time collector that makes the stolen tokens available to the attacker. A phishing page hosted on Cloudflare Pages mimics a legitimate Microsoft login interface to initiate the OAuth flow. When victims interact with the page, they are redirected to a localhost URL containing the authorization code. They are then tricked into pasting or dragging the URL back into the phishing page, triggering the automated data exfiltration pipeline.

Delivery, Post-Exploitation, and Mitigation

Attackers deploy personalized phishing emails generated from the harvested employee data. These emails frequently contain malicious links embedded inside PDF files hosted on DocSend to improve credibility and bypass spam filtering. Once the tokens are captured, they are imported into Specter Portal, enabling attackers to interact with compromised Microsoft environments. This grants access to permitted resources such as email, files, and other account-tied services. Push Security researchers emphasize that mitigation is structurally complicated because trust in first-party apps is deeply architectural. While the Family of Client IDs framework provides some defensive utility, administrators should implement token binding to trusted devices, establish behavioral detection rules, and enforce app authentication restrictions. EXEC Executive Overview

The threat intelligence digest covering the May 2, 2026 reporting window highlights a sustained shift toward highly automated identity-centric attacks, with the ConsentFix v3 campaign emerging as a primary indicator of this trend. This malicious operation specifically targets Microsoft Azure environments by leveraging OAuth authorization flows to harvest refresh tokens and achieve persistent access. The campaign’s technical execution centers on a novel phishing mechanism that subverts traditional credential harvesting. Instead of relying on static credential theft, ConsentFix v3 delivers customized emails directing targets to a malicious landing page. The page prompts users to paste or drag a localhost URL containing a valid OAuth authorization code. Once submitted, the captured code is immediately routed to an automation framework built on Pipedream, which programmatically exchanges the authorization code for a long-lived refresh token. This real-time token acquisition pipeline enables attackers to maintain access without requiring additional user interaction or bypassing interactive authentication prompts.

Infrastructure deployment within the ConsentFix v3 campaign demonstrates a clear emphasis on operational security and scalability. Threat actors utilize a distributed network of third-party hosting services to host phishing endpoints, reducing infrastructure fatigue and complicating traditional takedown efforts. The campaign also incorporates automated reconnaissance capabilities, harvesting employee details and organizational metadata to generate highly personalized phishing payloads. This contextual targeting increases click-through rates and reduces the likelihood of detection by standard security controls. Once the refresh tokens are successfully harvested, the compromised credentials are leveraged to authenticate against Microsoft services through the Specter Portal. The resulting session tokens effectively bypass multi-factor authentication requirements, as the OAuth flow completes successfully from the perspective of the identity provider. The underlying architectural trust model placed in first-party applications further complicates detection, as legitimate-looking token exchanges and API calls blend into normal Azure traffic patterns. The campaign’s reliance on serverless automation and cloud-native hosting reflects a broader industry trajectory where threat actors increasingly exploit legitimate development and integration platforms to orchestrate identity theft at scale.
