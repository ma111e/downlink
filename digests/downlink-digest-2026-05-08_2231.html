Overview and Attribution

Threat hunters have identified TCLBANKER, a previously undocumented Brazilian banking trojan tracked by Elastic Security Labs under the moniker REF3076. The malware family is evaluated as a significant evolution of the Maverick trojan and is attributed to the Water Saci threat cluster identified by Trend Micro. It is specifically designed to compromise 59 distinct banking, fintech, and cryptocurrency platforms.

Delivery and Evasion Mechanisms

The infection chain begins with a malicious MSI installer packaged inside a ZIP file. This installer abuses a legitimate signed Logitech executable known as Logi AI Prompt Builder to deploy a malicious DLL through side-loading techniques. The deployed screen_retriever_plugin.dll functions as a loader equipped with a comprehensive watchdog subsystem. This subsystem continuously scans for analysis tools, sandboxes, debuggers, disassemblers, instrumentation tools, and antivirus software.

The loader enforces strict execution conditions, only running if invoked by the legitimate Logitech process or a testing executable named tclloader.exe. It actively neutralizes endpoint security measures by replacing the ntdll.dll library to remove usermode hooks and disabling Event Tracing for Windows telemetry. To prevent premature execution in analysis environments, the malware generates an environment hash derived from anti-debugging checks, anti-virtualization checks, disk information, and system language verification. This hash is used to decrypt the embedded payload, and the presence of a debugger or incorrect language settings will cause decryption to fail and halt execution.

Persistence, Beaconing, and Capabilities

Once the environment checks pass, the main banking trojan component verifies the system locale and establishes persistence through a scheduled task. It then beacons to an external server using an HTTP POST request containing basic system telemetry. The malware features a self-update mechanism and a URL monitor that extracts the current address from the foreground browser. This monitor targets popular browsers including Google Chrome, Mozilla Firefox, Microsoft Edge, Brave, Opera, and Vivaldi.

When the extracted URL matches a hard-coded list of financial institutions, the trojan establishes a WebSocket connection to a remote command server. This connection enables a command dispatch loop that allows operators to perform extensive malicious activities. These capabilities include running shell commands, capturing screenshots, initiating screen streaming, manipulating the clipboard, launching a keylogger, and remotely controlling mouse and keyboard inputs. The trojan can also manage files and processes, enumerate running tasks, list visible windows, and serve fake credential-stealing overlays.

Social Engineering and Credential Harvesting

Data theft is primarily conducted through a Windows Presentation Foundation-based full-screen overlay framework. This framework orchestrates real-time social engineering by displaying credential harvesting prompts, vishing wait screens, bogus progress bars, and fake Windows Update notifications. A critical feature of these overlays is their ability to hide from screen capture tools, preventing automated analysis of the deception. The trojan maintains a continuous command loop over the WebSocket connection to dynamically adjust these social engineering tactics based on the victim’s interactions.

Worm Propagation and Distribution

The loader simultaneously activates a worming module designed to propagate the trojan at scale through spam and phishing campaigns. This module employs a two-pronged distribution approach leveraging WhatsApp Web and Microsoft Outlook. The WhatsApp component hijacks authenticated browser sessions and utilizes the open-source WPPConnect project to automate message delivery. It retrieves messaging templates from the command server and filters its targets to exclude groups, broadcasts, and non-Brazilian phone numbers.

The Outlook component operates as an email spambot that abuses the victim’s installed Microsoft Outlook application. By sending phishing emails directly from the victim’s own email address, the malware bypasses traditional spam filters and leverages the inherent trust of the compromised account. This distribution strategy effectively renders traditional email gateways and reputation-based defenses ineffective.

Strategic Implications

Elastic Security Labs concludes that TCLBANKER reflects a broader maturation within the Brazilian banking trojan ecosystem. Techniques that were once exclusive to highly sophisticated threat actors are now being packaged into accessible commodity crimeware. The campaign’s distribution model inherits the trust and deliverability of legitimate communications by hijacking existing WhatsApp sessions and Outlook accounts. This approach ensures that the malware spreads efficiently while evading conventional security controls.
