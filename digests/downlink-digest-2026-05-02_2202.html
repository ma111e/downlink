Executive Summary & Threat Evolution

The npm ecosystem has transitioned from a nuisance era of isolated typosquatting to a high-consequence threat landscape characterized by systematic, self-replicating supply chain attacks. The September 2025 Shai-Hulud worm marked this inflection point by proving that npm could be weaponized as a force multiplier for malware distribution. Campaign Details

Two major campaigns were tracked in April 2026. The first, identified as Shai-Hulud: The Third Coming, compromised the @bitwarden/cli package at version 2026.4.0. The second wave, dubbed Mini Shai-Hulud, targeted the SAP developer ecosystem through four compromised packages: @cap-js/sqlite@2.2.2, @cap-js/postgres@2.2.2, @cap-js/db-service@2.10.1, and mbt@1.2.48. These SAP packages account for approximately 570,000 weekly downloads and target enterprise developers with access to cloud credentials and deployment secrets. Both campaigns share identical toolchains, obfuscation methods, and propagation logic, indicating direct authorship reuse.

Attack Mechanism & Payload Capabilities

Compromised packages introduce two new files, setup.mjs and execution.js, alongside a modified package.json that adds a preinstall lifecycle hook. This hook triggers automatically during npm install, bootstrapping the attack before installation completes. The setup.mjs script detects the host operating system and architecture, then downloads the Bun JavaScript runtime (v1.3.13) from GitHub. It immediately executes the main payload using this runtime. The obfuscated payload, weighing approximately 11.7 MB, employs a custom string scrambling layer labeled ctf-scramble-v2 to evade static analysis. It also includes a Russian locale killswitch that exits silently if the system locale is set to Russian. On non-CI machines, the malware daemonizes itself to run persistently in the background.

Credential Harvesting & Exfiltration

The payload deploys multiple provider classes to scan and extract sensitive data from developer environments. It targets file systems for SSH keys, Git configurations, and .npmrc files. It runs shell commands to capture active GitHub CLI tokens and harvests full environment variable blocks. The malware also integrates cloud SDKs to enumerate and read secrets from AWS Systems Manager, Azure Key Vault, and Google Cloud Secret Manager. In CI/CD environments, it parses GitHub Actions runner contexts to extract masked secrets. All harvested data is compressed, encrypted with AES-256-GCM, and exfiltrated through a dual-channel system. The primary channel sends encrypted JSON payloads via HTTPS to an attacker-controlled C2 domain. The secondary channel creates public GitHub repositories with randomized Dune-themed names to commit the encrypted results.

Propagation & C2 Resilience

The malware implements an aggressive self-replication mechanism that backdoors every npm package the victim can publish. It validates npm tokens for write permissions, enumerates writable packages, downloads their tarballs, and injects the malicious setup.mjs script. It then bumps the patch version and republishes the package to the registry. This creates an exponential propagation vector that infects downstream developers. Command and control resilience is maintained through a GitHub dead-drop mechanism. If the primary C2 server is unreachable, the malware searches public GitHub commits for specific keywords. It decodes embedded Base64 strings to recover new C2 domains or previously stolen tokens, ensuring continuous operation without relying on traditional attacker-controlled infrastructure.

Mitigations & Recommendations

Security teams should immediately disable lifecycle scripts by adding ignore-scripts=true to .npmrc files. Organizations must enforce cooldown periods of 24 to 72 hours for newly published packages to catch malicious releases before widespread adoption. Dependency pinning should be enforced using package-lock.json and npm ci commands to prevent hidden updates during builds. All package downloads must be routed through private registry proxies rather than direct connections to the public registry. Namespace shadowing should be prevented by using scoped packages and configuring private registries to resolve internal scopes exclusively. Provenance verification should be implemented using tools like slsa-verifier to validate OpenID Connect attestations. Finally, strict egress network policies must be applied to CI runners to block unauthorized outbound connections to C2 infrastructure.

Indicators of Compromise

Network indicators include the C2 domain audit.checkmarx.cx, the IP address 94.154.172.43, and the attacker-controlled domain checkmarx.cx. GitHub indicators feature the dead-drop repository helloworm00/hello-world, commit SHA1 hash bc544f455d7c06c8a1f3446160a6d9a4a8236b11, and public repositories named with Dune-themed words. File indicators include SHA256 hashes for the bootstrap script bw_setup.js, the obfuscated payload bw1.js, and the malicious package.json. Process indicators highlight unexpected Bun runtime execution and the injection of .github/workflows/format-check.yml on transient branches.
