Incident Overview

A malicious Hugging Face repository impersonating OpenAI’s Privacy Filter project distributed an information-stealing malware payload to Windows users. The repository briefly reached the number one position on the platform’s trending list and accumulated 244,000 downloads before being removed following user reports. Hugging Face serves as a platform for developers and researchers to share AI models, datasets, and machine learning tools. The malicious repository exploited this ecosystem by hosting a pre-trained AI system that combined weight files, configuration data, and malicious code.

Discovery and Impersonation Tactics

Researchers at HiddenLayer discovered the campaign on May 7 after identifying a repository named Open-OSS/privacy-filter. The threat actors typosquatted OpenAI’s legitimate Privacy Filter release by copying its model card nearly verbatim. The repository shipped a loader.py file that was designed to fetch and execute infostealer malware on Windows machines. This Python script included fake AI-related code to appear harmless during initial inspection.

Malware Execution Chain

The loader.py script operated by disabling SSL verification and decoding a base64-encoded URL pointing to an external resource. It then fetched and executed a JSON payload containing a PowerShell command. This command executed in an invisible window to download a batch file named start.bat. The batch file performed privilege escalation, downloaded the final payload known as sefirah, added it to Microsoft Defender’s exclusions, and executed it. The final payload is a Rust-based infostealer designed to harvest sensitive data from the compromised system.

Targeted Data and Exfiltration

The infostealer targeted the following sensitive information:

  • Browser data from Chromium- and Gecko-based browsers, including cookies, saved passwords, encryption keys, browsing data, and session tokens
  • Discord tokens, local databases, and master keys
  • Cryptocurrency wallets and wallet browser extensions
  • SSH, FTP, and VPN credentials and configuration files, including FileZilla
  • Sensitive local files and wallet seeds or keys
  • System information and multi-monitor screenshots

All stolen data was compressed and exfiltrated to a command-and-control server located at recargapopular[.]com. The malware also incorporated extensive anti-analysis features to evade detection. These features included checks for virtual machines, sandboxes, debuggers, and analysis tools.

Campaign Metrics and Related Infrastructure

The exact number of victims remains unclear due to suspected artificial inflation of engagement metrics. Researchers noted that the vast majority of the 667 accounts that liked the repository appear to be auto-generated. The 244,000 download count was also likely inflated through similar automated means. By examining these metrics, researchers uncovered other repositories that utilized the same malicious loader infrastructure. HiddenLayer also identified technical overlaps with an npm typosquatting campaign that distributed the WinOS 4.0 implant.

Remediation Recommendations

Users who downloaded files from the malicious repository are advised to take immediate action. They should reimage their machines to ensure complete removal of the threat. All stored credentials must be rotated to prevent unauthorized access. Cryptocurrency wallets and seed phrases should be replaced to protect digital assets. Finally, browser sessions and tokens must be invalidated to block persistent access. EXEC Executive Overview

During the reporting window, threat intelligence platforms flagged a coordinated supply chain compromise leveraging the Hugging Face ecosystem to distribute a sophisticated infostealer. A malicious repository impersonating OpenAI’s Privacy Filter project was published on the platform, exploiting the high download counts and community engagement metrics typically associated with legitimate artificial intelligence tooling. The repository was subsequently removed after detection, but not before its loader script successfully executed on Windows endpoints. The initial execution chain relied on PowerShell commands to download and instantiate a Rust-based infostealer payload. This payload is engineered to harvest sensitive data across multiple attack surfaces, including browser session cookies, Discord authentication tokens, cryptocurrency wallet seed phrases, and local system credentials. Upon collection, the malware compresses the exfiltrated data and transmits it to a dedicated command-and-control infrastructure.

The infostealer demonstrates advanced operational security, employing anti-analysis techniques designed to bypass static and dynamic detection mechanisms. Researchers noted that the repository’s elevated download metrics and positive community feedback were likely inflated through auto-generated accounts, indicating a deliberate strategy to exploit platform reputation systems and lower the barrier to initial execution. This campaign reflects a broader trend of threat actors targeting developer-centric platforms and AI-related projects to distribute malware under the guise of legitimate open-source utilities. By leveraging the perceived trust associated with high-profile AI organizations like OpenAI, attackers are successfully circumventing traditional perimeter defenses and user scrutiny.

The technical execution highlights the continued evolution of Rust-based malware, which increasingly replaces traditional scripting languages due to its performance characteristics and native anti-debugging capabilities. The PowerShell loader serves as a lightweight bootstrap mechanism, prioritizing stealth and rapid payload deployment. Command-and-control communications are structured to compress and encrypt harvested artifacts before transmission, reducing network footprint and evading bandwidth-based detection thresholds. The use of platform reputation manipulation, combined with multi-stage delivery and robust evasion techniques, underscores a maturing threat landscape where supply chain integrity and platform verification mechanisms are primary vectors for initial access and data exfiltration.
