Vulnerability Overview and Classification

The U.S. Cybersecurity and Infrastructure Security Agency (CISA) has officially added CVE-2026-31431, commonly referred to as Copy Fail, to its Known Exploited Vulnerabilities (KEV) catalog. This designation follows confirmed evidence of active exploitation in the wild. The vulnerability is classified as a local privilege escalation (LPE) flaw with a CVSS score of 7.8. It allows an unprivileged local user to bypass standard security controls and obtain root-level access. The defect is nine years old and was introduced through three individually harmless modifications to the Linux kernel in 2011, 2015, and 2017.

Technical Mechanism and Exploitation

Copy Fail originates from a logic bug within the Linux kernel’s authentication cryptographic template. Exploit developers can reliably trigger privilege escalation using a compact 732-byte Python script. The attack functions by corrupting the kernel’s in-memory page cache of any readable file, which includes setuid binaries. This process effectively alters executables at execution time without ever touching the disk. Because the corruption occurs in memory, attackers can inject code into privileged binaries like /usr/bin/su to gain root privileges. The vulnerability impacts Linux distributions that have been shipped since 2017.

Impact on Containerized Environments

The widespread adoption of Linux in cloud infrastructure significantly amplifies the risk posed by this flaw. Security researchers at Kaspersky and Google-owned Wiz have highlighted that containerized environments face a heightened threat. Docker, LXC, and Kubernetes automatically grant processes inside a container access to the AF_ALG subsystem by default. This default configuration allows the Copy Fail exploit to breach container isolation and seize control over the physical host machine. The attack does not require advanced techniques such as memory address guessing or race conditions, substantially lowering the barrier to entry for attackers.

Detection Challenges and Proof-of-Concept Availability

Identifying this attack presents a significant challenge for security teams. The exploit relies exclusively on legitimate system calls, making it nearly indistinguishable from normal application behavior. A fully working proof-of-concept (PoC) is already available to the public. Researchers have also detected Go and Rust versions of the original Python implementation circulating in open-source repositories. Microsoft Defender Security Research Team has reported preliminary testing activity, suggesting that threat actors are likely to increase exploitation efforts in the coming days.

Mitigation Recommendations

The attack vector is strictly local and requires only low privileges without any user interaction. Although it cannot be exploited remotely in isolation, it becomes highly impactful when chained with initial access vectors like SSH compromise, malicious CI job execution, or existing container footholds. CISA has advised Federal Civilian Executive Branch (FCEB) agencies to apply kernel updates by May 15, 2026. If patching cannot be implemented immediately, organizations are recommended to implement the following controls:

  • Disable the affected cryptographic feature
  • Enforce network isolation
  • Apply strict access controls These measures provide a temporary defense until kernel patches in versions 6.18.22, 6.19.12, and 7.0 can be deployed.
